[Политики Azure](/azure/azure-policy/) позволяют обеспечить соответствие всех ресурсов, входящих в подписку, стандартам организации. Используйте политики, чтобы снизить затраты, разрешив развертывание только для утвержденных типов ресурсов и номеров SKU. Определите правила и действия для ресурсов, и эти правила будут автоматически применяться во время развертывания. Например, можно контролировать, какие типы ресурсов развертываются. Или же можно разрешить только утвержденные расположения для ресурсов. Одни политики позволяют запретить действие, а другие — настроить аудит действий.

Политики можно использовать с управлением доступом на основе ролей (RBAC). RBAC определяет доступ пользователей и представляет собой систему запрета по умолчанию и явного разрешения. Политика определяет свойства ресурсов во время и после их развертывания. Политика представляет собой систему разрешения по умолчанию и явного запрета.

При работе с политиками нужно знать две важные концепции: *определения* и *назначения политик*. Определение политики описывает условия управления, которые требуется применить. Назначение политики приводит определение в действие для определенной области.

![Назначение политик](./media/resource-manager-governance-policy/policy-concepts.png)

В Azure предоставляется несколько встроенных определений политик, которые можно использовать без изменений. Вы передаете значения параметров, чтобы указать значения, допустимые в вашей области. Если встроенное определение политики не соответствует вашим требованиям, вы можете [создать пользовательские определения политик](../articles/azure-policy/create-manage-policy.md).
