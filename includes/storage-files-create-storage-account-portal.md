Учетная запись хранения — это общий пул для хранения, в котором можно развернуть файловые ресурсы Azure или другие ресурсы хранения, например большие двоичные объекты или очереди. Учетная запись хранения может содержать неограниченное число файловых ресурсов. В общем ресурсе может храниться любое число файлов, насколько это позволяет емкость учетной записи хранения.

Чтобы создать учетную запись хранения, сделайте следующее:

1. В меню слева выберите **+**, чтобы создать ресурс.
2. Введите в поле поиска **учетная запись хранения**, выберите в результатах поиска **Учетная запись хранения — большой двоичный объект, файл, таблица, очередь** и нажмите кнопку **Создать**.
    ![Снимок экрана: запись учетной записи хранения должна выглядеть, как в диалоговом окно поиска ресурсов](../articles/storage/files/media/storage-how-to-use-files-portal/create-storage-account-1.png)

3. В поле **Имя** вводите *mystorageacct* с несколькими случайными числами, пока не появится зеленый значок, означающий, что это уникальное имя. Имя учетной записи хранения следует указывать в нижнем регистре. Оно должно быть глобально уникальным. Запишите имя учетной записи хранения. Оно понадобится вам позже. 
4. В поле **Модель развертывания** оставьте значение по умолчанию — **Resource Manager**. См. дополнительные сведения о [развертывании с помощью Azure Resource Manager и классическом развертывании (сведения о моделях развертывания и состоянии ресурсов)](../articles/azure-resource-manager/resource-manager-deployment-model.md).
5. В поле **Тип учетной записи** выберите **StorageV2**. См. дополнительные сведения о [разных типах учетных записей хранения Azure](../articles/storage/common/storage-account-options.md?toc=%2fazure%2fstorage%2ffiles%2ftoc.json).
6. В поле **Производительность** оставьте значение по умолчанию **Хранилище уровня "Стандартный"**. Служба файлов Azure сейчас поддерживает хранилище уровня Standard. Даже если выбрать хранилище Azure класса Premium, файловый ресурс будет храниться в стандартном хранилище.
7. В поле **Репликация** выберите **локально избыточное хранилище (LRS)**. 
8. В поле **Требуется безопасное перемещение** рекомендуем всегда выбирать **Включено**. Дополнительные сведения об этом параметре см. в статье [Требование безопасной передачи в службе хранилища Azure](../articles/storage/common/storage-require-secure-transfer.md?toc=%2fazure%2fstorage%2ffiles%2ftoc.json).
9. В разделе **Подписка** выберите подписку, в которой будет создана учетная запись хранения. Если у вас только одна подписка, она должна быть подпиской по умолчанию.
10. Для параметра **Группа ресурсов** выберите **Создать**. В поле "Имя" введите *myResourceGroup*.
11. В поле **Расположение** выберите **Восточная часть США**.
12. В поле **Виртуальные сети** оставьте вариант по умолчанию **Отключено**. 
13. Чтобы учетную запись можно было легко найти, выберите **Закрепить на панели мониторинга**.
14. Затем нажмите кнопку **Создать**, чтобы начать развертывание.
