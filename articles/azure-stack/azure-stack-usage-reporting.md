# <a name="report-azure-stack-usage-data-to-azure"></a>Передача в Azure данных об использовании из Azure Stack 

Данные об использовании, также называемые данными о потреблении, представляют собой объем используемых ресурсов. 

Многоузловые системы Azure Stack, применяющие модель выставления счетов на основе потребления ресурсов, должны отправлять данные об использовании в Azure для выставления счетов.  Настройка экземпляра Azure Stack для отправки данных об использовании в Azure осуществляется операторами Azure Stack.

> [!NOTE]
> Отправка данных о потреблении является обязательным пунктом для многоузловых систем Azure Stack, для которых применяется модель оплаты по мере использования. При этом она необязательна для пользователей, использующих модель на основе емкости (см. страницу о том, [как приобрести Azure Stack](https://azure.microsoft.com/en-us/overview/azure-stack/how-to-buy/)). Для пользователей Пакета средств разработки Azure Stack операторы могут передавать данные о потреблении и тестировать эту возможность. Тем не менее с них не взимается плата за использование. 


![поток выставления счетов](media/azure-stack-usage-reporting/billing-flow.png)

Данные об использовании отправляются из Azure Stack в Azure через мост Azure. В Azure коммерческая система обрабатывает данные об использовании и создает счет. После создания счета владелец подписки Azure может просмотреть и загрузить его в [Центре управления учетной записью Azure](https://account.windowsazure.com/Subscriptions). Чтобы узнать, как лицензируется Azure Stack, см. [документ с информацией о пакетах и ценах на Azure Stack](https://go.microsoft.com/fwlink/?LinkId=842847&clcid=0x409).

## <a name="set-up-usage-data-reporting"></a>Настройка передачи данных об использовании

Чтобы настроить передачу данных о потреблении, необходимо [зарегистрировать экземпляр Azure Stack в Azure](azure-stack-register.md). В рамках процесса регистрации компонент моста Azure Stack настраивается для подключения Azure Stack к Azure и отправки данных о потреблении. Из Azure Stack в Azure отправляются следующие данные о потреблении:

- **Идентификатор единицы измерения** — уникальный идентификатор использованного ресурса.
- **Количество** — объем используемых ресурсов.
- **Расположение** — расположение развертывания текущего ресурса Azure Stack.
- **URI ресурса** — полный универсальный код ресурса, об использовании которого передаются данные.
- **идентификатор подписки** — идентификатор подписки пользователя Azure Stack. Это локальная подписка (Azure Stack).
- **Время** — время начала и окончания передачи данных об использовании. Между потреблением ресурсов в Azure Stack и передачей данных о потреблении в коммерческую систему происходит некоторая задержка. Azure Stack вычисляет данные об использовании каждые 24 часа, а передача данных об использовании в конвейер коммерческой системы в Azure занимает еще несколько часов. Таким образом использование, происходящее около полуночи, может отобразиться в Azure на следующий день.

## <a name="generate-usage-data-reporting"></a>Формирование отчетов касательно данных о потреблении

1. Чтобы протестировать передачу данных об использовании, создайте несколько ресурсов в Azure Stack. Например, можно создать [учетную запись хранения](azure-stack-provision-storage-account.md), [виртуальную машину Windows Server](azure-stack-provision-vm.md) и Linux с базовым и стандартным номером SKU, чтобы увидеть, каким образом передаются данные об использовании ядер. Данные об использовании различных типов ресурсов передаются в различных единицах измерения.

2. Запустите ресурсы и дайте им поработать несколько часов. Сведения об использовании собираются примерно один раз в час. После сбора эти данные передаются в Azure и обрабатываются в коммерческой системе Azure. Этот процесс может занять до нескольких часов.

## <a name="view-usage---csp-subscriptions"></a>Просмотр использования для подписок CSP

Зарегистрировав Azure Stack с помощью подписки CSP, вы можете просматривать данные об использовании и затратах точно также же, как вы это делаете в Azure. Стоимость использования Azure Stack будет включена в ваш счет и указана в сверочном файле, доступном в [Центре партнеров](https://partnercenter.microsoft.com/partner/home). Сверочный файл обновляется ежемесячно. Если вам необходимо получить доступ к последней информации о потреблении Azure Stack, вы можете использовать API Центра партнеров.

   ![Центр партнеров](media/azure-stack-usage-reporting/partner-center.png)


## <a name="view-usage--enterprise-agreement-subscriptions"></a>Просмотр использования для подписок по Соглашению Enterprise

Зарегистрировав Azure Stack с помощью подписки по Соглашению Enterprise, вы можете просматривать данные об использовании и затратах на [портале EA](https://ea.azure.com/). Отчет об использовании Azure Stack можно будет загрузить вместе с данными об использовании Azure в разделе отчетов на портале EA. 

## <a name="view-usage--other-subscriptions"></a>Просмотр использования для других подписок

Зарегистрировав Azure Stack с помощью других подписок, например с оплатой по мере использования, вы можете просматривать данные об использовании и расходы в Центре управления учетной записью Azure. Войдите в [Центр управления учетной записью Azure](https://account.windowsazure.com/Subscriptions) с правами администратора учетной записи Azure и выберите подписку Azure, которую необходимо использовать для регистрации Azure Stack. Вы можете просмотреть данные об использовании в Azure Stack, сумму к оплате за каждый из используемых ресурсов, как показано на следующем рисунке:

   ![поток выставления счетов](media/azure-stack-usage-reporting/pricing-details.png)

При использовании Пакета средств разработки Azure Stack плата за ресурсы не взимается, поэтому отображается цена 0,00 долл. Когда многоузловые ресурсы Azure Stack станут общедоступными, вы сможете увидеть фактическую стоимость для каждого из них.

## <a name="which-azure-stack-deployments-are-charged"></a>За какие ресурсы, развернутые в Azure Stack, взимается плата?

Ресурсы в рамках Пакета средств разработки Azure Stack являются бесплатными. В то время как за использование многоузловых систем Azure Stack, виртуальных машин рабочих нагрузок, служб хранения и служб приложений плата взимается.

## <a name="are-users-charged-for-the-infrastructure-vms"></a>Взимается ли с пользователей плата за виртуальные машины инфраструктуры?

Нет. Данные об использовании для некоторых виртуальных машин поставщика ресурсов Azure Stack передаются в Azure, но плата за них не взимается, как и для виртуальных машин инфраструктуры Azure Stack, созданных во время развертывания.  

С пользователей взимается плата только за виртуальные машины, которые выполняются в подписках клиентов. Таким образом все рабочие нагрузки необходимо развернуть в подписках клиентов в соответствии с условиями лицензирования Azure Stack.

## <a name="i-have-a-windows-server-license-i-want-to-use-on-azure-stack-how-do-i-do-it"></a>Как использовать имеющуюся лицензию Windows Server в Azure Stack?

Применение имеющейся лицензии позволяет избежать создания единиц измерения использования. Имеющиеся лицензии Windows Server можно использовать в Azure Stack, как описано в разделе "Using existing software with Azure Stack" (Использование имеющегося программного обеспечения в Azure Stack) [руководства по лицензированию Azure Stack](https://go.microsoft.com/fwlink/?LinkId=851536&clcid=0x409). Пользователям необходимо развернуть виртуальные машины Windows Server, как описано в статье [Преимущество гибридного использования Azure для Windows Server](https://docs.microsoft.com/azure/virtual-machines/windows/hybrid-use-benefit-licensing), чтобы использовать имеющиеся лицензии.

## <a name="which-subscription-is-charged-for-the-resources-consumed"></a>В каких подписках взимается плата за использованные ресурсы?
Плата взимается в подписке, предоставляемой при [регистрации Azure Stack в Azure](azure-stack-register.md).

## <a name="what-types-of-subscriptions-are-supported-for-usage-data-reporting"></a>В каких типах подписок поддерживается передача данных об использовании?

При использовании многоузловых систем Azure Stack поддерживаются подписки по Соглашению Enterprise (EA) и подписки CSP. При использовании Пакета средств разработки Azure Stack передача данных об использовании поддерживается в подписках Enterprise Agreement (EA), подписке с оплатой по мере использования, CSP и MSDN.

## <a name="does-usage-data-reporting-work-in-sovereign-clouds"></a>Работает ли передача данных о потреблении в независимых облаках?

Для передачи данных о потреблении в Пакете средств разработки Azure Stack необходимы подписки, созданные в глобальной системе Azure. Подписки, созданные в одном из независимых облаков (Azure для государственных организаций, Azure для Германии или Azure для Китая), невозможно зарегистрировать в Azure, поэтому они не поддерживают передачу данных об использовании.

## <a name="how-can-users-identify-azure-stack-usage-data-in-the-azure-billing-portal"></a>Как пользователи могут определить данные об использовании Azure Stack на портале управления счетами и подписками Azure?

Пользователи могут просмотреть данные об использовании Azure Stack в файле со сведениями о потреблении. Чтобы узнать о том, как получить этот файл, см. раздел [Скачивание данных об использовании в Центре управления учетной записью (в формате CSV)](https://docs.microsoft.com/azure/billing/billing-download-azure-invoice-daily-usage-date#download-usage-from-the-account-center-csv). Файл со сведениями об использовании содержит единицы измерения Azure Stack, определяющие хранилище и виртуальные машины Azure Stack. Данные о всех ресурсах, используемых в Azure Stack, отправляются в области с именем "Azure Stack".

## <a name="why-doesnt-the-usage-reported-in-azure-stack-match-the-report-generated-from-azure-account-center"></a>Почему данные об использовании в Azure Stack не соответствуют данным, созданным в Центре управления учетной записью Azure?

Между данными об использовании, передаваемыми API-интерфейсами использования Azure Stack, и данными об использовании, передаваемыми Центром управления учетной записью Azure, всегда возникает задержка. Эта задержка обуславливается временем, необходимым для отправки данных об использовании из Azure Stack в коммерческую систему Azure. Из-за этой задержки использование, происходящее около полуночи, может отобразиться в Azure на следующий день. Если вы используете [API-интерфейсы использования Azure Stack](azure-stack-provider-resource-api.md) и сравните результаты с данными о потреблении на портале управления счетами и подписками Azure, то можете увидеть различие.
