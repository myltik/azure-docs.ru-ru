# <a name="manage-cost-of-data-retention-with-your-log-analytics-workspace"></a>Управление затратами на хранение данных с помощью рабочей области Log Analytics
При регистрации в Log Analytics, в зависимости от выбранного плана, действует ограничение времени хранения данных, создаваемых в подключенных источниках в рабочей области.  В данной статье описываются факторы, которые могут повлиять на стоимость хранения этих данных в течение различных периодов времени, и приводятся инструкции по настройке этого ограничения.   

Так как Log Analytics может обрабатывать большие объемы данных из источников в локальной, облачной и гибридных средах, затраты на хранение данных в течение определенного периода времени могут быть пропорционально велики. Это зависит от следующих факторов:

* количество систем, компонентов инфраструктуры, облачных ресурсов, т. д., из которых собираются данные;
* тип данных, создаваемых в источнике, таких как очереди сообщений, журналы, события, связанные с безопасностью данные или метрики производительности;
* тома данных, создаваемые этими источниками; 
* количество включенных решений по управлению и источников данных, а также частота сбора данных.

> [!NOTE]
> Обратитесь к документации по каждому решению. В ней должна быть приведена оценка объема данных, которые оно собирает.   

Если вы используете план *Бесплатный*, то данные будут храниться семь дней.  Для уровня *Автономный* или *Платный* доступны собранные данные за последний 31 день.  

Если при использовании плана *Бесплатный* вы обнаружите, что постоянно превышаете допустимые объемы, то сможете изменить рабочую область и выбрать платный план, чтобы собирать данные свыше этого ограничения. 

> [!NOTE]
> Если выбрать более длительный срок хранения для платного уровня, это приведет к увеличению затрат. Вы можете изменить тип плана в любое время. Дополнительные сведения о ценах см. в разделе [Цены на Log Analytics](https://azure.microsoft.com/pricing/details/log-analytics/). 

## <a name="change-the-data-retention-period"></a>Изменение срока хранения данных 

1. Войдите на [портал Azure](http://portal.azure.com). 
2. Щелкните **Больше служб** в нижнем левом углу. В списке ресурсов введите **Log Analytics**. Как только вы начнете вводить символы, список отфильтруется соответствующим образом. Выберите **Log Analytics**.
3. В области подписок Log Analytics выберите из списка рабочую область для изменения.
4. На странице рабочей области щелкните **Хранение** в левой области.
5. В области "Хранение" рабочей области воспользуйтесь ползунком, чтобы увеличить или уменьшить количество дней, а затем нажмите кнопку **Сохранить**.  Если вы используете уровень *Бесплатный*, то вы не сможете изменить срок хранения данных. Необходимо перейти на платный уровень, чтобы управлять этим параметром.<br><br> ![Изменение параметра хранения данных рабочей области](media/log-analytics-manage-cost/manage-cost-change-retention.png)
