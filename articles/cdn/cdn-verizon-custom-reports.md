# <a name="custom-reports-from-verizon"></a>Пользовательские отчеты из Verizon

[!INCLUDE[cdn-verizon-only](../../includes/cdn-verizon-only.md)]

Используя пользовательские отчеты Verizon через портал управления для профилей Verizon, можно определить тип данных, собираемых для отчетов о пограничных записях CNAME.


## <a name="accessing-verizon-custom-reports"></a>Доступ к пользовательским отчетам Verizon
1. В колонке профиля сети CDN нажмите кнопку **Управление** .
   
    ![Кнопка "Управление" для профиля CDN](./media/cdn-reports/cdn-manage-btn.png)
   
    Откроется портал управления CDN.
2. Наведите указатель на вкладку **Аналитика**, а затем — на всплывающий элемент **Пользовательские отчеты**. Щелкните **Edge CNAMEs** (Пограничные записи CNAME).
   
    ![Портал управления CDN — меню "Пользовательские отчеты"](./media/cdn-reports/cdn-custom-reports.png)

## <a name="edge-cnames-custom-report"></a>Пользовательский отчет о пограничных записях CNAME
Пользовательский отчет о пограничных записях CNAME содержит статистику попаданий и переданных данных для пограничных записей CNAME, для которых включено ведение журнала для пользовательских отчетов. Пограничные записи CNAME состоят из имен узлов конечных точек сети доставки содержимого Azure и любых связанных имен узлов личных доменов. 

Ведение журнала данных для пользовательского отчета начинается через один час после включения функции создания отчетов о пограничных записях CNAME. Данные отчета можно просмотреть, создав отчет о пограничных записях CNAME для конкретной платформы или для всех платформ. Данный отчет будет охватывать только пограничные записи CNAME, для которых данные пользовательского отчета были собраны во время заданного периода времени. Отчет о пограничных записях CNAME состоит из диаграммы и таблица данных для первых 10 записей CNAME согласно метрике, определенной с помощью параметра "Метрика". 

Создайте пользовательский отчет, определив следующие параметры отчета.

- "Метрики". Поддерживаются следующие параметры:

   - "Попадания". Указывает общее число запросов, направленных к пограничной записи CNAME, для которой включена функция пользовательских отчетов. Эта метрика не включает в себя код состояния, возвращаемый клиенту.

   - "Переданные данные". Указывает общий объем данных, переданных с пограничных серверов в HTTP-клиенты (например, веб-браузеры) для запросов, направленных к пограничной записи CNAME, для которой включена функция создания пользовательских отчетов. Объем переданных данных вычисляется путем добавления заголовков ответов HTTP в текст ответа. В результате этого объем данных, переданных для каждого ресурса, превышает фактический размер его файла.

- "Groupings" (Группировка). Определяет тип статистически, отображаемой под линейчатой диаграммой. Поддерживаются следующие варианты.

   - "HTTP Response Codes" (Коды ответа HTTP). Позволяет упорядочить статистику по коду ответа HTTP (например, 200, 403 и т. д.), возвращаемого клиенту. 

   - "Состояние кэша". Позволяет упорядочить статистику по состоянию кэша.


Чтобы задать диапазон дат для отчета, можно выбрать из раскрывающегося списка предопределенный диапазон дат, например **Сегодня** или **За эту неделю**, или щелкнуть **Настраиваемый** и вручную ввести диапазон дат, щелкнув значки календаря. 

После выбора диапазона дат нажмите кнопку **Перейти**, чтобы создать отчет.

Можно экспортировать данные в формате Excel, щелкнув значок Excel справа от кнопки **Перейти**.

## <a name="edge-cnames-custom-report-fields"></a>Поля пользовательского отчета о пограничных записях CNAME

| Поле                     | ОПИСАНИЕ   |
|---------------------------|---------------|
| "2xx"                       | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является код состояния HTTP, 2xx (например, "200 ОК"). |
| "3xx"                       | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является код состояния HTTP, 3xx (например, "302 - Объект найден" или "304 - Объект не изменялся"). |
| "4xx"                       | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является код состояния HTTP, 4xx (например, "400 - Неверный запрос", "403 Запрещено" или "404 Не найдено"). |
| "5xx"                       | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является код состояния HTTP, 5xx (например, "500 - Внутренняя ошибка сервера" или "502 - Неверный шлюз"). |
| "Cache Hit %" (% попаданий в кэш)               | Указывает процент кэшированных запросов, обработанных для запрашивающей стороны напрямую из кэша. |
| Попаданий в кэш                | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является попадание в кэш (например, CP_EXPIRED_HIT, TCP_HIT или TCP_PARTIAL_HIT). Попадание в кэш происходит при обнаружении кэшированной версии запрошенного содержимого. |
| "Data Transferred (MB)" (Переданные данные (МБ))     | Указывает общий объем данных (МБ), переданных с пограничных серверов в HTTP-клиенты (веб-браузеры) для пограничной записи CNAME. Объем переданных данных вычисляется путем добавления заголовков ответов HTTP в текст ответа. В результате этого объем данных, переданных для каждого ресурса, превышает фактический размер его файла. |
| ОПИСАНИЕ               | Идентифицирует пограничную запись CNAME по ее имени узла. |
| Попадания                      | Указывает общее число запросов к пограничной записи CNAME. |
| "Промахи"                    | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является промах кэша (например, TCP_CLIENT_REFRESH_MISS, TCP_EXPIRED_MISS или TCP_MISS). Промах кэша происходит, если запрашиваемое содержимое не было сохранено в кэше на пограничном сервере, который обработал запрос. | 
| "Без кэша"                  | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является код состояния кэша CONFIG_NOCACHE.  |
| Другие                     | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является код состояния HTTP вне диапазона 2xx–5xx. |
| платформа                  | Указывает платформу, которая обрабатывает трафик пограничной записи CNAME. |
| "Не присвоено"               | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, для которой сведения о коде состояния кэша или коде состояния HTTP не были добавлены в журнал.  |
| "Uncacheable" (Некэшируемые)               | Указывает общее количество запросов или переданных данных (МБ) для пограничной записи CNAME, результатом которых является код состояния кэша UNCACHEABLE.  |


## <a name="considerations"></a>Рекомендации
Отчеты могут формироваться только за последние 18 месяцев.
