# <a name="understand-how-the-reserved-virtual-machine-instance-discount-is-applied"></a>Сведения о применении скидки на зарезервированный экземпляр виртуальной машины
После того как вы купили зарезервированный экземпляр виртуальной машины, скидка резервирования автоматически применяется к виртуальным машинам, атрибуты и количество которых совпадают с резервированием. Резервирование покрывает затраты на инфраструктуру виртуальных машин. В приведенной ниже таблице описаны затраты на виртуальную машину после приобретения резервирования. Во всех случаях с вас взимается плата по обычным тарифам на хранение и сеть.

| Тип виртуальной машины  | Расходы с резервированием |    
|-----------------------|--------------------------------------------|
|Виртуальные машины Linux без дополнительного программного обеспечения | Резервирование покрывает затраты на инфраструктуру вашей виртуальной машины.|
|Виртуальные машины Linux с платой за программное обеспечение (например, Red Hat) | Резервирование покрывает затраты на инфраструктуру. Взимается плата за дополнительное программное обеспечение.|
|Виртуальные машины Windows без дополнительного программного обеспечения |Резервирование покрывает затраты на инфраструктуру. Взимается плата за дополнительное программное обеспечение Windows.|
|Виртуальные машины Windows с дополнительным программным обеспечением (например, SQL Server) | Резервирование покрывает затраты на инфраструктуру. Взимается плата за программное обеспечение Windows, а также за дополнительное программное обеспечение.|
|Виртуальные машины Windows с [преимуществом гибридного использования Azure](https://docs.microsoft.com/azure/virtual-machines/windows/hybrid-use-benefit-licensing) | Резервирование покрывает затраты на инфраструктуру. Затраты на программное обеспечение Windows покрываются преимуществом гибридного использования Azure. Все дополнительное программное обеспечение оплачивается отдельно.| 

## <a name="application-of-reservation-discount-to-non-windows-vms"></a>Применение скидки резервирования для виртуальных машин с другими ОС
 Скидка резервирования применяется к работающим экземплярам виртуальных машин на почасовой основе. Для применения скидки приобретенное резервирование должно соответствовать потреблению ресурсов работающими виртуальными машинами. Для виртуальных машин, использующихся меньше часа, резервирование будет заполнено из других виртуальных машин, которые его не используют, включая одновременно работающие виртуальные машины. По окончании часа приложение резервирования для виртуальных машин блокируется. Если виртуальная машина работает неполный час или одновременно работающие виртуальные машины в течение часа не занимают весь час резервирования, резервирование становится недостаточным за этот час. Приведенная ниже диаграмма иллюстрирует резервирование экземпляров виртуальной машины с оплачиваемым использованием. На диаграмме представлено одно оплаченное резервирование и два соответствующих экземпляра виртуальных машин.

![Применение зарезервированного экземпляра виртуальной машины](media/billing-reserved-vm-instance-application/billing-reserved-vm-instance-application.png)

1.  Ресурсы, потребляемые свыше определенных для зарезервированного экземпляра виртуальной машины, оплачиваются по мере использования. Если вы используете меньше ресурсов, чем предоставляет оплаченный зарезервированный экземпляр виртуальной машины, дополнительная плата не взимается.
2.  В течение первого часа экземпляр 1 работает 0,75 часа, а экземпляр 2 работает 0,5 часа. Общее использование в течение 1 часа — 1,25 часа. За оставшиеся 0,25 часа с вас взимается оплата по мере использования.
3.  В течение второго и третьего часов оба экземпляра работали 1 час. Работа одного экземпляра покрывается резервированием, а за второй взимается оплата по мере использования.
4.  В течение четвертого часа экземпляр 1 работает 0,5 часа, а экземпляр 2 работает 1 час. Использование экземпляра 1 полностью покрываться резервированием. 0,5 часа работы экземпляра 2 также покрываются. За оставшиеся 0,5 часа с вас взимается оплата по мере использования.

Чтобы больше узнать о применении резервирования и просмотреть отчеты о начислении платы за использование, см. статью об [использовании зарезервированного экземпляра виртуальной машины](https://go.microsoft.com/fwlink/?linkid=862757).

## <a name="application-of-reservation-discount-to-windows-vms"></a>Применение скидки резервирования для виртуальных машин Windows
Если используются экземпляры виртуальных машин Windows, резервирование покрывает затраты на инфраструктуру. Резервирование применяется к затратам на инфраструктуру виртуальных машин Windows так же, как и для виртуальных машин с другими ОС. За программное обеспечение Windows плата взимается отдельно для каждого виртуального процессора. См. статью о [затратах на программное обеспечение Windows с резервированием](https://go.microsoft.com/fwlink/?linkid=862756). Вы можете покрыть расходы на лицензирование Windows, используя [Преимущество гибридного использования Azure для Windows Server] (https://docs.microsoft.com/azure/virtual-machines/windows/hybrid-use-benefit-licensing).
