# <a name="understand-azure-spending-limit-and-how-to-remove-it"></a>Общие сведения о предельной сумме расходов Azure и ее отключении

Предельная сумма расходов в Azure существует, чтобы избежать превышения суммы по кредиту. Для всех новых клиентов, которые зарегистрировали бесплатную пробную версию или приобрели на несколько месяцев предложение с деньгами на счете, по умолчанию установлена предельная сумма расходов. Она составляет 0 долл. США. Ее невозможно изменить. Предельная сумма расходов не используется для подписок с оплатой по мере использования и тарифных планов с фиксированной ежемесячной платой. Список всех доступных предложений Azure и сведения о доступности предельной суммы расходов см. [здесь](https://azure.microsoft.com/support/legal/offer-details/).

**Сведения об оповещениях о выставлении счетов** см. в статье [Настройка оповещений о выставлении счетов или предоставлении кредитов для подписок Microsoft Azure](billing-set-up-alerts.md).

## <a name="what-happens-when-i-reach-the-spending-limit"></a>Что происходит при достижении предельной суммы расходов

Если вы превысили ежемесячную сумму, предусмотренную для вашей подписки, ваши развернутые службы отключаются до конца расчетного периода во избежание дополнительных расходов. 

Например, если вы использовали всю кредитную сумму по вашей подписке, развернутые облачные службы удаляются из рабочей среды, виртуальные машины Azure останавливаются и выделенные для них ресурсы освобождаются. Данные в учетных записях хранения и базах данных будут доступны только в режиме для чтения.

Если для вашей подписки предусмотрены деньги на счете на несколько месяцев, то в начале следующего расчетного периода ваша подписка будет снова активирована автоматически. После этого вы сможете повторно развернуть свои облачные службы и получить полный доступ к своим учетным записям хранения и базам данных.

При достижении предельной суммы расходов для подписки мы отправляем электронные уведомления. Войдите в [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions), чтобы просмотреть уведомления о подписках, в которых достигнута предельная сумма расходов.

Если вы используете бесплатную пробную версию и достигли предельной суммы расходов, вы можете [перейти на предложение с оплатой по мере использования](billing-upgrade-azure-subscription.md), чтобы снять ограничения на предельную сумму расходов и автоматически активировать подписку.

## <a name="remove-the-spending-limit-in-account-center"></a>Снятие ограничения на предельную сумму расходов в Центре управления учетной записью

Вы можете снять ограничение предельной суммы расходов в любой момент при условии, что в вашей подписке указан действительный метод оплаты. Для предложений с деньгами на счете на несколько месяцев это ограничение можно активировать повторно. Ограничение будет установлено в начале следующего периода выставления счетов.

Чтобы снять ограничение, выполните следующие действия:

1. Войдите в [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions).
1. Выберите подписку.
1. Если подписка отключена из-за превышения установленной суммы расходов, щелкните уведомление "Для подписки достигнут расходный лимит, и она была отключена для предотвращения расходов". В противном случае в области **Состояние подписки** щелкните **Снять предельную сумму расходов**.
1. Выберите нужный вам вариант.

|Параметр|Результат|
|-------|-----|
|Удалить предельную сумму расходов на неопределенный срок|Удаление предельной суммы расходов без ее автоматического включения в начале следующего расчетного периода.|
|Удалить предельную сумму расходов для текущего расчетного периода|Удаление предельной суммы расходов с ее автоматическим включением в начале следующего расчетного периода.|

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

### <a name="why-would-i-want-to-remove-the-spending-limit"></a>Зачем снимать ограничение на предельную сумму расходов?

Из-за установленной предельной суммы расходов у вас могут возникнуть проблемы с развертыванием или использованием некоторых сторонних служб, а также служб Майкрософт. Ниже описаны ситуации, в которых в подписке следует снять ограничение на предельную сумму расходов.

* Вы планируете развернуть образы от таких поставщиков, как Oracle, или таких служб, как Visual Studio Team Services. В этом случае предельная сумма расходов будет превышена почти мгновенно, и ваша подписка будет отключена.
* У вас есть службы, работу которых нельзя прерывать.
* У вас есть службы или ресурсы с определенными параметрами (например, виртуальные IP-адреса), которые вы не хотите терять. При освобождении ресурсов и служб такие параметры сбрасываются.

### <a name="how-do-i-turn-on-the-spending-limit-after-removing-it"></a>Как включить ограничение предельной суммы расходов, если оно было снято?

Эта функция доступна, только если предельная сумма расходов удалена без возможности восстановления. Настройте автоматическое включение этой функции в начале следующего периода выставления счетов.

1. Войдите в [Центр управления учетной записью](https://account.windowsazure.com/Subscriptions).
1. Щелкните желтый баннер, чтобы изменить настройку для предельной суммы расходов.
1. Выберите **Включить функцию предельной суммы расходов для следующего периода выставления счетов \<дата начала расчетного периода\>**.

### <a name="how-do-i-set-a-custom-spending-limit"></a>Как задать настраиваемую предельную сумму расходов?

Сейчас нет возможности установить настраиваемую предельную сумму расходов. Тем не менее вы можете подписаться на [получение предупреждений касательно выставления счетов для контроля ваших затрат](billing-set-up-alerts.md).

### <a name="does-the-spending-limit-prevent-all-charges-from-azure"></a>Влияет ли предельная сумма расходов на все расходы в Azure?

За использование некоторых [служб Майкрософт и сторонних служб](billing-understand-your-azure-marketplace-charges.md) с вашего счета будут списываться средства, даже если установлена предельная сумма расходов. Примером таких служб могут быть лицензии на Visual Studio, служба Azure Active Directory Premium, планы поддержки и большинство сторонних служб, продаваемых в [Azure Marketplace](https://azure.microsoft.com/marketplace/).
