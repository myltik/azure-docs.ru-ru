# <a name="supported-data-sources-for-azure-machine-learning-data-preparation"></a>Поддерживаемые источники данных в инструменте подготовки данных службы "Машинное обучение Azure" 
В этой статье приведены источники данных, поддерживаемые инструментом подготовки данных службы "Машинное обучение Azure".

Поддерживаемые в этом выпуске источники данных приведены ниже.

## <a name="types"></a>Типы 

### <a name="sql-server"></a>SQL Server;
Чтение с локального сервера SQL или из базы данных Azure SQL.

#### <a name="options"></a>Параметры
- Адрес сервера
- доверенный сервер (даже если сертификат на сервере недействителен; следует использовать с осторожностью);
- тип аутентификации (Windows, сервер);
- Имя пользователя
- Пароль
- база данных для подключения;
- Запросы SQL

#### <a name="notes"></a>Заметки
- Столбцы sql-variant не поддерживаются.
- Столбец времени преобразуется в тип datetime. Для этого время из базы данных добавляется к дате 01.01.1970.
- При выполнении в кластере Spark во всех столбцах, связанных с датами (date, datetime, datetime2, datetimeoffset), будут вычислены неправильные значения для дат до значения 1583.
- Точность значений в столбцах для десятичных чисел может понизиться из-за преобразования.

### <a name="directory-vs-file"></a>Каталог или файл
Выберите отдельный файл и передайте его в инструмент подготовки данных, который проанализирует его тип и определит параметры подключения файла по умолчанию, отображающиеся на следующей странице.

Выберите каталог или набор файлов в каталоге (средство выбора файлов поддерживает множественный выбор). При этом файлы считываются в виде отдельного потока данных и добавляются друг к другу (при необходимости с отделенными заголовками).

Далее приведены поддерживаемые типы файлов:
- с разделителями (CSV, TSV, TXT и т. д.);
- с полями фиксированного размера;
- с обычным текстом;
- Файл JSON

### <a name="csv-file"></a>CSV-файл
Чтение файла со значениями с разделителями-запятыми из хранилища.

#### <a name="options"></a>Параметры
- Разделитель
- Комментарий
- Заголовки
- Десятичный символ.
- Кодировка файла.
- Пропускаемые строки.

### <a name="tsv-file"></a>TSV-файл
Чтение файла со значениями с разделением знаками табуляции из хранилища.

#### <a name="options"></a>Параметры
- Комментарий
- Заголовки
- Кодировка файла.
- Пропускаемые строки.

### <a name="excel-xlsxlsx"></a>Excel (XLS и XLSX)
Чтение файла Excel (по одному листу) с помощью указания имени номера листа.

#### <a name="options"></a>Параметры
- Номер или имя листа.
- Заголовки
- Пропускаемые строки.

### <a name="json-file"></a>Файл JSON
Чтение файла JSON из хранилища. Файл преобразовывается в плоскую структуру при чтении.

#### <a name="options"></a>Параметры
- None

### <a name="parquet"></a>PARQUET
Чтение набора данных Parquet (одного файла или папки).

Формат PARQUET может принимать разные формы в хранилище. Иногда для наборов данных меньшего размера используется один PARQUET-файл. Различные библиотеки Python поддерживают чтение или запись в одном PARQUET-файле. Сейчас инструмент подготовки данных службы "Машинное обучение Azure" выполняет чтение файла Parquet во время локального интерактивного использования с помощью библиотеки Python PyArrow. Эта библиотека поддерживает отдельные файлы (если они написаны в таком виде, а не входят в состав большего набора данных) и наборы данных PARQUET.

Набор данных PARQUET — это коллекция файлов PARQUET, каждый из которых представляет более мелкую секцию набора данных большего размера. Наборы данных обычно содержатся в папке. Это стандартный формат выходных данных PARQUET для платформ, таких как Spark и Hive.

>[!NOTE]
>При чтении данных PARQUET, находящихся в папке с несколькими файлами PARQUET, безопаснее всего выбрать каталог для чтения и установить флажок **Parquet Dataset** (Набор данных PARQUET). Поэтому PyArrow считывает всю папку, а не отдельные файлы, обеспечивая тем самым поддержку чтения более сложных способов хранения PARQUET на диске, например, секционирование папки.

Горизонтальное масштабирование зависит от возможностей чтения Parquet Spark. Этот процесс поддерживает отдельные файлы и папки (аналогично локальному интерактивному использованию).

#### <a name="options"></a>Параметры
- Parquet data set (Набор данных PARQUET). Этот параметр определяет, раскрывает ли инструмент подготовки данных службы "Машинного обучения Azure" каталог и выполняет ли попытку чтения каждого файла отдельно (в режиме при снятом флажке), или он рассматривает каталог как полный набор данных (в режиме с активированным флажком). В режиме с активированным флажком PyArrow выбирает самый оптимальный способ интерпретации файлов.


## <a name="locations"></a>Расположения
### <a name="local"></a>Local
Локальный жесткий диск или сопоставленное сетевое расположение для хранения.

### <a name="sql-server"></a>SQL Server;
Локальный сервер SQL или база данных Azure SQL.

### <a name="azure-blob-storage"></a>Хранилище больших двоичных объектов Azure
Хранилище BLOB-объектов Azure, для которого требуется подписка Azure.
