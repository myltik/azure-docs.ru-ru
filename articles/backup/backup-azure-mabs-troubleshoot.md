# <a name="troubleshoot-azure-backup-server"></a>Устранение неполадок Azure Backup Server

Используйте информацию в таблицах ниже для устранения ошибок, обнаруженных в ходе использования Azure Backup Server.

## <a name="invalid-vault-credentials-provided"></a>Указаны недопустимые учетные данные хранилища 

Для решения этой проблемы выполните такие [действия по устранению неполадок](https://docs.microsoft.com/azure/backup/backup-azure-mabs-troubleshoot#registration-and-agent-related-issues).

## <a name="the-agent-operation-failed-because-of-a-communication-error-with-the-dpm-agent-coordinator-service-on-the-server"></a>Операция агента завершилась сбоем из-за ошибки связи со службой координатора агента DPM на сервере 

Для решения этой проблемы выполните такие [действия по устранению неполадок](https://docs.microsoft.com/azure/backup/backup-azure-mabs-troubleshoot#registration-and-agent-related-issues). 

## <a name="setup-could-not-update-registry-metadata"></a>При установке не удалось обновить метаданные реестра

Для решения этой проблемы выполните такие [действия по устранению неполадок](https://docs.microsoft.com/azure/backup/backup-azure-mabs-troubleshoot#installation-issues).

Настройте сервер DPM для [ретрансляции по протоколу SMTP на O365 с помощью служб IIS](https://technet.microsoft.com/library/aa995718(v=exchg.65).aspx).<br><br> **Важное примечание.** Обязательно используйте формат user@domain.com *вместо* domain\user.<br><br><li>Укажите DPM использовать имя локального сервера в качестве сервера SMTP (порт 587). Затем укажите для него адрес электронной почты пользователя, который будет использоваться для отправки сообщений.<li> Имя пользователя и пароль на странице настройки DPM SMTP должны быть учетной записью домена в домене, где размещен DPM. </li><br> **Примечание.** При изменении адреса SMTP-сервера измените новые параметры, закройте окно параметров, а затем снова откройте его, чтобы убедиться, что новое значение отражается.  При изменении и проверке новые параметры не всегда задействуются. Так что лучше всего проверять именно так.<br><br>Удалить параметры можно в любое время. Для этого закройте консоль DPM и измените следующие разделы реестра: **HKLM\SOFTWARE\Microsoft\Microsoft Data Protection Manager\Notification\ <br/> Удалите разделы SMTPPassword и SMTPUserName**. Их можно добавить в пользовательском интерфейсе при его запуске.
