# <a name="upgrade-to-vm-backup-stack-v2"></a>Обновление до стека резервного копирования виртуальных машин версии 2
Стек резервного копирования виртуальных машин версии 2 содержит следующие улучшения функций.
* Возможность просмотра моментального снимка, сделанного в рамках задания резервного копирования. Его можно использовать для восстановления, не дожидаясь завершения передачи данных.
Таким образом не нужно будет дожидаться окончания копирования моментального снимка в хранилище до активации восстановления. Кроме того, это позволит исключить дополнительные требования к хранилищу касательно резервного копирования виртуальных машин типа "Премиум", за исключением первой резервной копии.  

* Сокращение времени резервного копирования и восстановления за счет локального сохранения моментальных снимков в течение 7 дней. 

* Поддержка дисков размером до 4 ТБ.  

* Возможность использования оригинальных учетных записей хранения (даже когда виртуальная машина имеет диски, распределенные между учетными записями хранения) при восстановлении неуправляемой виртуальной машины. Это ускорит восстановление для самых разных конфигураций виртуальных машин. 
    > [!NOTE] 
    > Это не то же самое, что переопределение исходной виртуальной машины. 
    > 
    >

## <a name="what-is-changing-in-the-new-stack"></a>Что изменилось в новом стеке?
В настоящее время задание резервного копирования состоит из двух этапов:
1.  Создание моментального снимка виртуальной машины. 
2.  Передача моментального снимка виртуальной машины в хранилище резервных копий Azure. 

Точка восстановления считается созданной только после выполнения этапов 1 и 2. Точка восстановления в рамках нового стека создается сразу после создания моментального снимка. Кроме того, из нее можно выполнить восстановление, используя ту же последовательность восстановления. Эту точку восстановления можно определить на портале Azure, указав "моментальный снимок" в качестве типа точки восстановления. Когда моментальный снимок переносится в хранилище, тип точки восстановления изменяется на "Моментальный снимок и хранилище". 

![Задание резервного копирования в стеке резервного копирования виртуальных машин версии 2](./media/backup-azure-vms/instant-rp-flow.jpg) 

По умолчанию моментальные снимки хранятся в течение семи дней. Это позволяет быстрее завершить восстановление из этих моментальных снимков за счет сокращения времени, необходимого для копирования данных из хранилища в учетную запись хранения. 

## <a name="considerations-before-upgrade"></a>Рекомендации перед обновлением
* Это однонаправленное обновление стека резервного копирования виртуальных машин. Таким образом, все будущие резервные копии попадут в этот поток. Так как **стек включен на уровне подписки, все виртуальные машины попадут в этот поток**. Все новые добавления функций будут основаны на том же стеке. Возможность контролировать добавление функций на уровне политики будет реализована в будущих выпусках. 
* Для виртуальных машин с дисками типа "Премиум" во время первого резервного копирования до его завершения убедитесь, что в учетной записи хранения доступно пространство хранения, эквивалентное размеру виртуальной машины. 
* Так как моментальные снимки хранятся локально, чтобы оптимизировать создание точки восстановления и ускорить восстановление, вы увидите соответствующие расходы на хранение моментальных снимков в течение семидневного периода.
* Добавочные моментальные снимки хранятся как страничные BLOB-объекты. Со всех клиентов, использующих неуправляемые диски, будет взиматься плата за 7 дней хранения моментальных снимков в локальной учетной записи хранения клиента. В соответствии с текущей моделью ценообразования клиенты, использующие управляемые диски, не несут каких-либо дополнительных затрат.
* При выполнении восстановления из точки восстановления моментального снимка для виртуальной машины типа "Премиум" вы увидите, что расположение временного хранилища используется, пока виртуальная машина создается в рамках восстановления. 
* В случае учетных записей хранения уровня "Премиум" моментальные снимки, создаваемые для мгновенного восстановления, будут занимать 10 ТБ пространства, выделенного в учетной записи хранения уровня "Премиум".

## <a name="how-to-upgrade"></a>Обновление
### <a name="the-azure-portal"></a>Портал Azure
При использовании портала Azure можно увидеть уведомление на панели мониторинга хранилища, связанное с поддержкой большого диска и улучшением скорости резервного копирования и восстановления.

![Задание резервного копирования в стеке резервного копирования виртуальных машин версии 2](./media/backup-azure-vms/instant-rp-banner.png) 

Чтобы открыть экран для обновления до нового стека, выберите баннер. 

![Задание резервного копирования в стеке резервного копирования виртуальных машин версии 2](./media/backup-azure-vms/instant-rp.png) 

### <a name="powershell"></a>PowerShell
Выполните следующие командлеты из терминала PowerShell с повышенными привилегиями:
1.  Войдите в учетную запись Azure. 

```
PS C:> Connect-AzureRmAccount
```

2.  Выберите подписку, которую вы хотите зарегистрировать для использования предварительной версии.

```
PS C:>  Get-AzureRmSubscription –SubscriptionName "Subscription Name" | Select-AzureRmSubscription
```

3.  Зарегистрируйте эту подписку для использования закрытой предварительной версии.

```
PS C:>  Register-AzureRmProviderFeature -FeatureName "InstantBackupandRecovery" –ProviderNamespace Microsoft.RecoveryServices
```

## <a name="verify-whether-the-upgrade-is-complete"></a>Проверка завершения обновления
Из терминала PowerShell с повышенными привилегиями запустите следующий командлет:

```
Get-AzureRmProviderFeature -FeatureName "InstantBackupandRecovery" –ProviderNamespace Microsoft.RecoveryServices
```

Если появится сообщение "Registered" (Зарегистрировано), то подписка будет обновлена до стека резервного копирования виртуальных машин версии 2.
