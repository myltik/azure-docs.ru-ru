# <a name="how-to-use-the-azure-marketplace-seller-insights-portal"></a>Как использовать портал Azure Marketplace Seller Insights

В этой статье описано, как администраторы осуществляют доступ к порталу Seller Insights, добавляют или удаляют пользователей, а также задают для них разрешения.

## <a name="onboard-users"></a>Подключение пользователей

В этом разделе показано, как в первый раз получить доступ к порталу Seller Insights с правами администратора. Задачи в этом разделе предназначены для администраторов. 


### <a name="overview-of-roles"></a>Общие сведения о ролях

Администратор портала Seller Insights может добавлять и удалять пользователей, а также изменять их разрешения.

Пользователь портала Seller Insights может просматривать все данные, доступные вашей организации.

Администраторы и пользователи должны проходить проверку подлинности с помощью действительной [учетной записи Майкрософт](http://signup.live.com) либо рабочей или учебной учетной записи. Идентификатор новой учетной записи должен быть связан с отслеживаемым почтовым ящиком.

![Тип учетной записи][5]



### <a name="invitation-to-activate-your-account"></a>Приглашение для активации учетной записи

Перед входом на портал Seller Insights администратору необходимо определить учетные записи лиц, которых нужно сделать пользователями. Настраивайте пользователей, используя их рабочие или учебные учетные записи либо учетные записи Майкрософт.

Для активации службы администратор должен перейти по [URL-адресу](https://reports.azure.com), указанному в приглашении, полученному по электронной почте.

>[!NOTE]
>Если в качестве адреса электронной почты администратора указана учетная запись Майкрософт, а у вас нет учетной записи Майкрософт, связанной с адресом электронной почты, по которому вы получили приглашение, перед входом в систему [создайте необходимую учетную запись](https://signup.live.com).


### <a name="administrator-sign-in-to-the-seller-insights-portal"></a>Вход администратора на портал Seller Insights

1. Войдите на портал Seller Insights, щелкнув URL-адрес в сообщении, которое вы получили по электронной почте.
2. Введите адрес электронной почты и пароль администратора.
3. Нажмите кнопку **Войти**.
4. Начните администрирование учетной записи. Настройте разрешения, добавьте или измените пользователей.
5. Не забудьте предоставить пользователям URL-адрес портала Seller Insights.

    ![Страница входа][6]

## <a name="user-sign-in-to-the-portal"></a>Вход пользователя на портал

Администратор должен отправить вам сообщение о создании вашей учетной записи и предоставить вам [URL-адрес](https://reports.azure.com) для входа на портал Seller Insights.

>[!NOTE]
>Администратор — это владелец учетной записи, с помощью которой публикуются решения в Azure Marketplace. Владелец этой учетной записи может предоставлять доступ другим пользователям.  Если у вас есть доступ к порталу публикации с правами соадминистратора, адрес администратора можно найти, выполнив следующие действия:
>1. Войдите на портал публикации.
>2. Откройте вкладку **Издатели**.
>3. Откройте раздел нужного издателя (обычно он только один).
>4. Щелкните **Администраторы** в нижнем левом углу.
>5. Адрес электронной почты администратора, который вы ищете, будет указан как адрес *владельца*.


Вы можете войти на портал Seller Insights с помощью личной учетной записи Майкрософт либо рабочей или учебной учетной записи. Тип учетной записи зависит от того, какую учетную запись администратор использовал при настройке вашего профиля. Администратор должен сообщить вам, какую учетную запись следует использовать.

![Тип учетной записи][1]



**Добро пожаловать на портал Seller Insights!**

1. После входа откроется страница **Сводка**.
2. Ознакомьтесь с политикой конфиденциальности.
3. Вы в любое время можете **обратиться к сотрудникам службы поддержки** по вопросам, связанным с работой портала Seller Insights.
4. Для перемещения по порталу Seller Insights предназначены вкладки в верхней части страницы.  Вкладки в верхней части страниц в этом руководстве помогут вам разобраться с функциональностью портала Seller Insights.
5. Щелкните смайлик, чтобы в любое время отправить отзыв о портале Seller Insights.
6. Чтобы выйти с портала Seller Insights, щелкните свое имя в правом верхнем углу.

    ![Главная страница по умолчанию][2]


## <a name="summary-view"></a>Представление сводки


1. Щелкните вкладку **Сводка**.
2. Введите необходимую **дату начала** периода и щелкните **Применить**. Отобразятся данные за три месяца.
3. Вы увидите данные о тренде новых заказов за выбранный период (три месяца).
4. Новые заказы за выбранный период будут представлены с группированием по каналам и типам предложения.
5. Выберите представление на этой панели. **5 первых рынков**, **5 первых выполняющихся предложений** или **5 первых клиентов**.
6. Здесь будет показана схема тренда для оплаченных предложений за выбранный период времени. При наличии только предложений с использованием собственной лицензии или предложений, предлагаемых непосредственно Майкрософт, эта панель не отображается.

    ![Представление сводки][7]

## <a name="orders-and-usage"></a>Заказы и использование
### <a name="overview"></a>Обзор

На *панели A* можно выбрать вкладку **Заказы и использование**.  Здесь можно выбрать **Заказы** или **Использование** на вкладке **Тип отчета**, а также выбрать интересующие вас данные с помощью ползунков.

На *панели B* можно выбрать ежемесячные данные. Выберите **Заказы** или **Использование** в зависимости от выбора, сделанного на вкладке **Тип отчета** на панели A. Представленные данные будут соответствовать введенным датам и фильтрам, примененным с помощью ползунков на панели A.

На *панели C* можно выбрать данные о тренде за три месяца. Выберите **Заказы** или **Использование** в зависимости от выбора, сделанного на вкладке **Тип отчета** на панели A. Представленные данные будут соответствовать введенным датам и фильтрам, примененным с помощью ползунков на панели A.

На *панели D* представлены данные, связанные с каждой отдельной подпиской.  Данные на этой панели можно скачать в формате Excel или CSV.

### <a name="orders"></a>Заказы
#### <a name="monthly-view-panel-a"></a>Представление за месяц. Панель A

![Карта панели А][10]

  1. Перейдите на вкладку **Заказы и использование** в верхней строке для доступа к данным о заказах и использовании.
  2. Выберите **Заказы** на вкладке **Тип отчета**.
  3. Используйте символ фильтра, чтобы отобразить или скрыть срезы.  Используйте эти срезы для фильтрации данных: по **типу лицензии Azure**, **типу лицензии Marketplace**, **рынку**, **типу предложения** и состоянию **предварительной версии**.  Определения этих терминов содержатся на вкладке **Справка**.
  4. Выберите необходимую дату начала и нажмите кнопку **Перейти**.

  ![Представление заказов за месяц. Панель A][14]

#### <a name="monthly-view-panel-b"></a>Представление за месяц. Панель В

  ![Карта панели В][11]

  1. Выберите **Monthly View** (Представление за месяц).
  2. Отобразится поканальная сводка по заказам: CSP, прямое соглашение Enterprise, соглашение Enterprise через посредника и Web Direct.

* первые 10 заказов по рынкам;
* первые 10 заказов по типам предложений;
* заказы по типам лицензий Azure;
* заказы по типам лицензий Marketplace.


![Представление заказов за месяц. Панель В][15]

  ![Карта панели С][12]

  2. Выберите элементы, которые требуется просмотреть, в раскрывающемся списке **Предметная область** и нажмите кнопку **Применить**. Значение по умолчанию — **Выделить все**.

  5. Используйте символ **+**, чтобы переключиться в режим для отображения данных, на основе которых созданы диаграммы, выбранные для просмотра.

  ![Представление тренда заказов. Панель С][16]

## <a name="user-management"></a>Управление пользователями

Задачи в этом разделе предназначены для администраторов. Ниже приведены инструкции по добавлению и удалению пользователей, а также изменению их разрешений на портале Seller Insights.


### <a name="add-user"></a>Добавление пользователя

В этом разделе показано, как добавлять новых пользователей и изменять их разрешения на портале Seller Insights.


1. Перейдите на вкладку **Управление пользователями**.
2. Чтобы добавить пользователя, щелкните **Добавить пользователя**.
3. Введите адрес электронной почты пользователя, связанный с учетной записью Майкрософт либо с рабочей или учебной учетной записью. Затем нажмите кнопку **Продолжить**.
4. Администратор может назначить пользователям доступ к отчету с правами "только для чтения".
5. Введите данные пользователя (**Имя**, **Фамилия** и **Адрес электронной почты**) и нажмите кнопку **Продолжить**.
6. Выберите разрешения, которые нужно назначить пользователю, и нажмите кнопку **Продолжить**.
7. Проверьте введенные данные и нажмите кнопку **Сохранить**, чтобы продолжить.  Теперь пользователь успешно добавлен.

  ![Панель добавления пользователей][29]


### <a name="edit-user"></a>Изменение пользователя

1. Откройте вкладку **Управление пользователями**, а затем щелкните **Изменить пользователей**.
2. Найдите пользователя, разрешения которого нужно изменить.  Можно также добавить новых пользователей. Щелкните **Добавить пользователей**.
3. Щелкните разрешения, которые требуется удалить для пользователя.  Можно также удалить пользователя. Щелкните **Удалить пользователя**.
4. Чтобы продолжить, нажмите кнопку **Сохранить**.  Разрешения пользователя успешно изменены.

  ![Панель изменения пользователей][30]

## <a name="help-support-requests"></a>Справка по запросам в службу поддержки


### <a name="access-the-support-request-format"></a>Доступ к форме запроса в службу поддержки

1. Перейдите на вкладку **Справка**.
2. Выберите **Поддержка и документация**, а затем — **Начать запрос**.
3. Вы также можете отправить запрос на поддержку, воспользовавшись ссылкой **Свяжитесь с нами**. Эта ссылка отображается в нижней части каждой страницы в приложении.
4. Откроется новая вкладка, где вы сможете отправить запрос в службу поддержки.

  ![Доступ к форме запроса в службу поддержки][31]

### <a name="review-helpful-information"></a>Просмотр полезных сведений

1. Чтобы отправить в службу поддержки запрос, связанный с отчетами, выберите **Отчеты и аналитика** в раскрывающемся списке **Тип проблемы**.
2. Выберите категорию.  Доступные параметры: **Questions related to reports** (Вопросы, связанные с отчетами) и **Другое**.
3. Отобразятся полезные ссылки, которые могут помочь найти ответ на ваш запрос.  Просмотрите приведенную по этим ссылкам информацию, прежде чем отправлять запрос в службу поддержки.
4. Если вам все еще необходимо отправить запрос в службу поддержки, выберите **Начать запрос**.

  ![Панель изменения пользователей][32]


### <a name="enter-information-and-submit-the-request"></a>Ввод сведений и отправка запроса

1. Введите контактные данные, часовой пояс, регион и языковые параметры и нажмите кнопку **Продолжить**.
2. Дайте название проблеме и укажите как можно больше дополнительных сведений.  Также вы можете отправить вспомогательные файлы, например снимки экрана, после чего нажмите кнопку **Отправить**.
3. Вы получите подтверждение, что запрос был успешно отправлен.
4. Вы можете проверить состояние запроса в службу поддержки в любое время.

  ![Доступ к форме запроса в службу поддержки][33]


## <a name="reset-a-password"></a>Сброс пароля

**Проверка учетной записи и пароля**

1. *Проверьте, что учетная запись, которую вы используете, зарегистрирована и пароль не работает.* При вводе учетной записи, зарегистрированной в Azure Active Directory, вам будет автоматически предложено выбрать способ входа в систему. Используйте рабочую или учебную учетную запись либо учетную запись Майкрософт в качестве личной учетной записи.
2. Перейдите на веб-сайт по адресу [https://reports.azure.com](https://reports.azure.com) и выйдите из учетной записи, с помощью которой вы вошли в систему. Эту же страницу можно открыть в отдельном окне в конфиденциальном режиме. Таким образом другая учетная запись, с помощью которой вы уже вошли в систему, не будет учитываться.
3. Введите адрес электронной почты издателя, не вводя пароль. Если учетная запись зарегистрирована в Azure Active Directory, вы будете перенаправлены на шаг 4.

    >[!NOTE]
    >Учетные записи администраторов зарегистрированы. Также зарегистрированы рабочие или учебные учетные записи, настроенные с помощью учетной записи администратора.

4. Выберите **Рабочая или учебная учетная запись**. Рабочие или учебные учетные записи настраиваются компаниями, которые зарегистрированы в Azure. 

    >[!NOTE]
    >Возможна ситуация, когда пользователь настроил учетную запись Майкрософт, используя адрес электронной почты организации. Через некоторое время организация зарегистрировалась в Azure и тот же адрес электронной почты был зарегистрирован в качестве рабочей или учебной учетной записи. Этот пользователь может иметь учетную запись Майкрософт с тем же паролем, который использовался для создания рабочей или учебной учетной записи. Он также может иметь рабочую или учебную учетную запись с отдельным паролем, используемым в организации.

5. При выборе **Рабочая или учебная учетная запись** на портале Seller Insights обязательно используйте пароль рабочей или учебной учетной записи, а не пароль учетной записи Майкрософт.
6. После ввода рабочей или учебной учетной записи вы будете перенаправлены на страницу издателя для входа.
7. На этой странице проверьте, работает ли пароль рабочей или учебной учетной записи. Если пароль не работает, перейдите к разделу "Сброс пароля".



**Сброс паролей**

Для сброса пароля используйте учетные записи Майкрософт и рабочие или учебные учетные записи, которые указаны ниже.

>[!NOTE]
>Если администратор настроил вашу рабочую или учебную учетную запись и включил для нее возможность сброса пароля, вы можете сбросить свой пароль. В противном случае обратитесь к администратору через форму сброса пароля.

* [Учетная запись Майкрософт](https://account.live.com/ResetPassword.aspx)
![Password reset Microsoft account][3]

* [Рабочая или учебная учетная запись](https://passwordreset.microsoftonline.com/)
![Password reset work or school account][4]
