## <a name="alertlogic-log-manager"></a>AlertLogic Log Manager
AlertLogic Log Manager собирает журналы виртуальных машин, приложений и платформы Azure для анализа безопасности и хранения. Сюда входит сбор журналов аудита Azure через API Azure Monitor.  Эта информация используется для обнаружения несанкционированных операций и соответствия нормативным требованиям.

[Ознакомьтесь с документацией][alertlogic-doc]

## <a name="appdynamics"></a>AppDynamics
Служба управления производительностью приложений (APM) AppDynamics позволяет владельцам приложений быстро устранять узкие места, снижающие производительность, и оптимизировать производительность приложений, которые работают в среде Azure. AppDynamics APM легко интегрируется с Azure Marketplace и может быть использована для мониторинга облачных служб Azure (PaaS) (включая веб-роли и рабочие роли), виртуальных машин (IaaS), средств обнаружения удаленных служб (служебная шина Microsoft Azure), удаленных служб Azure очереди Microsoft Azure (большой двоичный объект Azure), очереди Azure (служебная шина (Майкрософт)), хранилища данных и хранилища BLOB-объектов Microsoft Azure.

[Ознакомьтесь с документацией][appdynamics-doc]

## <a name="cloudhealth"></a>CloudHealth
Объедините свое облако с платформой, которая автоматизирует его, позволив значительно сэкономить время и деньги. Непревзойденная видимость, интуитивная оптимизации и проверенные рекомендации по управлению CloudHealth позволят переопределить управление облаком. Платформа Cloudhealth позволяет предприятиям и MSP максимально повысить рентабельность инвестиций в облако и уверенно принимать решения о затратах, использовании, производительности и безопасности.

[Дополнительные сведения.][cloudhealth-doc]

## <a name="cloudmonix"></a>CloudMonix
CloudMonix предлагает службы мониторинга, автоматизации и самовосстановления для платформы Microsoft Azure.

[Ознакомьтесь с документацией][cloudmonix-doc]

## <a name="cloudyn"></a>Cloudyn
Cloudyn осуществляет управление и оптимизацию многоплатформенных, гибридных облачных развертываний, позволяя предприятиям полностью реализовать свой облачный потенциал. Это решение SaaS позволяет следить за использованием, производительностью и затратами, а также получать полезные сведения и практические рекомендации для интеллектуальной оптимизации и управления облачной инфраструктурой. Cloudyn обеспечивает учет с помощью точного взимания средств за использование и управления иерархическим распределением затрат. Решение Cloudyn интегрировано со службой мониторинга Azure, чтобы обеспечить полезные сведения и практические рекомендации для оптимизации развертывания Azure.

[Ознакомьтесь с документацией][cloudyn-doc]

## <a name="datadog"></a>Datadog
Datadog является ведущей мировой службой мониторинга для облачных приложений, которая объединяет данные с серверов, из баз данных, инструментов и служб, чтобы создать единое представление всего стека. Эти возможности предоставляются на платформе аналитики данных на основе SaaS, позволяющий командам специалистов по разработке и операциям работать совместно, чтобы избегать простоев, устранять проблемы производительности и обеспечивать своевременное завершение циклов разработки и развертывания. Благодаря интеграции Datadog и Azure можно собирать и просматривать метрики по инфраструктуре, сопоставлять метрики виртуальной машины с метриками уровня приложения, а также секционировать и фрагментировать метрики с помощью любой комбинации свойств и настраиваемых тегов.

[Ознакомьтесь с документацией][datadog-doc]

## <a name="dynatrace"></a>Dynatrace
Dynatrace OneAgent интегрируется с виртуальными машинами и службами приложений Azure посредством соответствующих механизмов расширений Azure.
Таким образом можно собирать метрики производительности узлов, сети и служб.
Помимо простого отображения метрик, мы можем отобразить комплексное представление сред, показывающая транзакции со стороны клиента на уровне базы данных.
Корреляция проблем на основе Application Insights и полностью интегрированный анализ первопричин, включая полезные сведения о коде и базах данных на уровне методов, намного упрощают устранение неполадок и оптимизацию производительности.

[Ознакомьтесь с документацией][dynatrace-doc]

## <a name="opsgenie"></a>OpsGenie
OpsGenie выполняет роль диспетчера оповещений, создаваемых Azure. OpsGenie определяет, кого нужно поставить в известность, исходя из дежурных расписаний и правил эскалации, и отправляет уведомляет их посредством электронной почты, текстовых сообщений (SMS), телефонных звонков или push-уведомлений. Проще говоря, Azure создает оповещения об обнаруженных проблемах, а OpsGenie гарантирует, что ими займутся соответствующие сотрудники.

[Ознакомьтесь с документацией][opsgenie-doc]

## <a name="pagerduty"></a>PagerDuty
PagerDuty, ведущее решение по управлению инцидентами, предоставило первоклассную поддержку оповещений на основе метрик Azure. Сейчас помимо уведомлений на основе метрик уровня платформы для служб Azure PagerDuty поддерживает уведомления на основе оповещений Azure Monitor, уведомлений автомасштабирования и событий журнала аудита. Эти усовершенствования позволяют пользователям лучше понять, как работает основная платформа Azure, и использовать все преимущества управления инцидентами в PagerDuty, чтобы получать ответы в режиме реального времени. Расширенная интеграция Azure возможна благодаря веб-перехватчикам webhook, обеспечивающим быструю и простую установку и настройку.

[Ознакомьтесь с документацией][pagerduty-doc]

## <a name="sciencelogic"></a>ScienceLogic
ScienceLogic предоставляет платформу контроля качества ИТ-услуг следующего поколения для управления любой технологией и в любом месте.  В одной платформе ScienceLogic обеспечивает возможности масштабирования, защиты, автоматизации и устойчивости, необходимые, чтобы упростить постоянно усложняющуюся задачу управления ИТ-ресурсами, службами и приложениями, которые находятся в постоянном движении.  Платформа ScienceLogic использует интерфейсы API Azure для взаимодействия с Microsoft Azure.  ScienceLogic дает возможность в реальном времени отслеживать службы и ресурсы Azure, чтобы вы видели, когда что-то перестало работать, и могли как можно скорее исправить проблему. Наряду с другими системами и службами центров обработки данных и облаков вы можете также управлять Azure.

[Подробнее][sciencelogic-doc]

## <a name="sumo-logic"></a>Sumo Logic
Sumo Logic — это безопасное облачное решение службы аналитики машинных данных, непрерывно в режиме реального времени предоставляющее аналитические данные из структурированных, полуструктурированных и неструктурированных данных на протяжении всего жизненного цикла приложения и во всем стеке. Более 1000 клиентов по всему миру используют Sumo Logic для анализа, а также для создания, выполнения, защиты современных приложений и облачных инфраструктур. Используя Sumo Logic, клиенты получают преимущества мультитенантной модели службы, ускоряют переход к непрерывному внедрению инноваций, увеличивают конкурентоспособность, а также ускоряют развитие бизнеса.

[Подробнее][sumologic-doc]
