# <a name="administer-group-policy-on-an-azure-ad-domain-services-managed-domain"></a>Администрирование групповой политики в управляемом домене доменных служб Azure AD
Доменные службы Azure Active Directory включают встроенные объекты групповой политики (GPO) для контейнеров "Пользователи AADDC" и "Компьютеры AADDC". Вы можете изменять эти встроенные объекты групповой политики, чтобы настроить групповую политику для управляемого домена. Кроме того, участники группы "Администраторы контроллера домена AAD" могут создавать собственные пользовательские подразделения в управляемом домене. Они также могут создавать пользовательские объекты групповой политики и связывать их с этими пользовательскими подразделениями. Пользователи, принадлежащие к группе "Администраторы контроллера домена AAD", получают права на администрирование групповой политики в управляемом домене.

## <a name="before-you-begin"></a>Перед началом работы
Чтобы выполнить задачи, описанные в этой статье, вам потребуется следующее:

1. Действующая **подписка Azure**.
2. **Каталог Azure AD** — синхронизированный с локальным каталогом или каталогом только для облака.
3. **Доменные службы Azure AD** должны быть включены для каталога Azure AD. Если это еще не сделано, выполните все задачи, описанные в [руководстве по началу работы](active-directory-ds-getting-started.md).
4. **Виртуальная машина, присоединенная к домену** , для администрирования домена, управляемого доменными службами Azure AD. Если у вас нет такой виртуальной машины, выполните все задачи, описанные в статье [Присоединение виртуальной машины Windows Server к управляемому домену](active-directory-ds-admin-guide-join-windows-vm.md).
5. Для администрирования групповой политики управляемого домена нужно иметь в каталоге учетные данные **учетной записи пользователя, входящего в группу "Администраторы контроллера домена AAD"**.

<br>

## <a name="task-1---provision-a-domain-joined-virtual-machine-to-remotely-administer-group-policy-for-the-managed-domain"></a>Задача 1. Подготовка присоединенной к домену виртуальной машины для удаленного администрирования групповой политики управляемого домена
Управляемыми доменами доменных служб Azure AD можно управлять удаленно с помощью привычных средств администрирования Active Directory, таких как центр администрирования Active Directory или AD PowerShell. Аналогичным образом групповую политику управляемого домена можно администрировать удаленно с помощью соответствующих средств.

Администраторы в каталоге Azure AD не имеют привилегий для подключения к контроллерам доменов в управляемом домене с помощью удаленного рабочего стола. Члены группы "Администраторы контроллера домена AAD" могут удаленно администрировать групповую политику для управляемых доменов. Для этого можно использовать средства групповой политики на компьютере под управлением Windows Server или клиентском компьютере, подключенном к нужному управляемому домену. Средства групповой политики можно установить в составе необязательного компонента "Управление групповой политикой" на серверы Windows Server и клиентские компьютеры, присоединенные к управляемому домену.

Первой задачей шагом является подготовка виртуальной машины Windows Server, которая присоединена к управляемому домену. Инструкции см. в статье [Присоединение виртуальной машины Windows Server к управляемому домену](active-directory-ds-admin-guide-join-windows-vm.md).

## <a name="task-2---install-group-policy-tools-on-the-virtual-machine"></a>Задача 2. Установка средств групповой политики на виртуальную машину
Чтобы установить средства администрирования групповой политики на присоединенную к домену виртуальную машину, выполните следующие действия.

1. Перейдите на портал Azure. Выберите **Все ресурсы** на панели слева. Найдите и выберите виртуальную машину, созданную в задаче 1.
2. Нажмите кнопку **Подключиться** на вкладке "Обзор". Будет создан и скачан файл протокола удаленного рабочего стола (RDP-файл).

    ![Подключение к виртуальной машине Windows](./media/active-directory-domain-services-admin-guide/connect-windows-vm.png)
3. Чтобы подключиться к виртуальной машине, откройте скачанный RDP-файл. При появлении запроса нажмите кнопку **Подключиться**. При запросе имени пользователя и пароля укажите учетные данные пользователя, принадлежащего к группе "Администраторы контроллера домена AAD". В нашем примере мы используем bob@domainservicespreview.onmicrosoft.com. При входе в систему может появиться предупреждение о сертификате. Чтобы продолжить процесс подключения, нажмите кнопку "Да" или "Продолжить".
4. На начальном экране откройте **диспетчер серверов**. На центральной панели окна диспетчера серверов щелкните **Добавить роли и компоненты** .

    ![Запуск диспетчера серверов на виртуальной машине](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager.png)
5. На странице **Перед началом работы** **мастера добавления ролей и компонентов** нажмите кнопку **Далее**.

    ![Страница "Перед началом работы"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-begin.png)
6. На странице **Тип установки** оставьте флажок **Установка ролей или компонентов** и нажмите кнопку **Далее**.

    ![Страница "Тип установки"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-type.png)
7. На странице **Выбор сервера** выберите текущую виртуальную машину из пула серверов и нажмите кнопку **Далее**.

    ![Страница "Выбор сервера"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-server.png)
8. На странице **Роли сервера** нажмите кнопку **Далее**. Мы пропустим эту страницу, так как не устанавливаем никакие роли на сервере.
9. На странице **Функции** выберите **Управление групповой политикой**.

    ![Страница "Компоненты"](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-gp-management.png)
10. На странице **Подтверждение** щелкните **Установить**, чтобы установить на виртуальную машину компонент "Управление групповой политикой". По завершении установки компонента нажмите кнопку **Закрыть**, чтобы выйти из мастера **добавления ролей и компонентов**.

    ![Страница подтверждения](./media/active-directory-domain-services-admin-guide/install-rsat-server-manager-add-roles-gp-management-confirmation.png)

## <a name="task-3---launch-the-group-policy-management-console-to-administer-group-policy"></a>Задача 3. Запуск и использование консоли управления групповой политикой
Для администрирования групповой политики управляемого домена можно использовать консоль управления групповой политикой на виртуальной машине, присоединенной к домену.

> [!NOTE]
> Администрировать групповую политику управляемого домена, могут участники группы "Администраторы контроллера домена AAD".
>
>

1. На начальном экране щелкните **Администрирование**. Должна открыться консоль **Управление групповой политикой**, установленная на виртуальной машине.

    ![Запуск управления групповыми политиками](./media/active-directory-domain-services-admin-guide/gp-management-installed.png)
2. Щелкните **Управление групповой политикой**, чтобы открыть консоль управления групповой политикой.

    ![Консоль управления групповой политикой](./media/active-directory-domain-services-admin-guide/gp-management-console.png)

## <a name="task-4---customize-built-in-group-policy-objects"></a>Задача 4. Настройка встроенных объектов групповой политики
Существуют два встроенных объекта групповой политики (GPO) — по одному для контейнеров "Компьютеры AADDC" и "Пользователи AADDC" в управляемом домене. Вы можете изменять эти объекты групповой политики, чтобы настроить групповую политику для управляемого домена.

1. В консоли **Управление групповой политикой** разверните узлы **Лес: contoso100.com** и **Домены**, чтобы просмотреть групповые политики для управляемого домена.

    ![Встроенные объекты групповой политики](./media/active-directory-domain-services-admin-guide/builtin-gpos.png)
2. Вы можете изменять эти встроенные объекты групповой политики, чтобы настроить групповые политики для управляемого домена. Щелкните правой кнопкой мыши объект групповой Политики и нажмите **Изменить**, чтобы настроить встроенный объект GPO. Для настройки GPO используется редактор конфигурации групповой политики.

    ![Редактирование встроенного объекта групповой политики](./media/active-directory-domain-services-admin-guide/edit-builtin-gpo.png)
3. Теперь с помощью консоли **редактора управления групповыми политиками** вы можете изменить встроенный объект GPO. На следующем снимке экрана представлен пример настройки встроенного объекта GPO "Компьютеры AADDC".

    ![Настройка объекта групповой политики](./media/active-directory-domain-services-admin-guide/gp-editor.png)

## <a name="task-5---create-a-custom-group-policy-object-gpo"></a>Шаг 5. Создание пользовательского объекта групповой политики
Можно создать или импортировать собственные пользовательские объекты групповой политики. Эти объекты также можно связать с пользовательским подразделением, созданным в управляемом домене. Дополнительные сведения о создании пользовательских подразделений см. в статье [Создание подразделения в управляемом домене доменных служб Azure AD](active-directory-ds-admin-guide-create-ou.md).

> [!NOTE]
> Администрировать групповую политику управляемого домена, могут участники группы "Администраторы контроллера домена AAD".
>
>

1. В консоли **Управление групповой политикой** щелкните, чтобы выбрать пользовательские организационное подразделение. Щелкните организационное подразделение правой кнопкой мыши и выберите пункт **Создать объект групповой политики в этом домене и связать его...**

    ![Создание пользовательского объекта групповой политики](./media/active-directory-domain-services-admin-guide/gp-create-gpo.png)
2. Укажите имя нового объекта групповой политики и нажмите кнопку **ОК**.

    ![Указание имени объекта групповой политики](./media/active-directory-domain-services-admin-guide/gp-specify-gpo-name.png)
3. Новый объект групповой политики создается и связывается с пользовательским подразделением. Щелкните правой кнопкой мыши объект групповой политики и выберите в меню команду **Изменить...**

    ![Созданный объект групповой политики](./media/active-directory-domain-services-admin-guide/gp-gpo-created.png)
4. Созданный объект групповой политики можно настроить с помощью **редактора "Управление групповыми политиками"**.

    ![Настройка нового объекта групповой политики](./media/active-directory-domain-services-admin-guide/gp-customize-gpo.png)


Дополнительные сведения об использовании [консоли управления групповой политикой](https://technet.microsoft.com/library/cc753298.aspx) можно найти в библиотеке Technet.
