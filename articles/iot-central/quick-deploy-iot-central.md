# <a name="create-an-azure-iot-central-application"></a>Создание приложения Azure IoT Central

С помощью пользовательского интерфейса Azure IoT Central _конструктор_ может определить свое приложение Microsoft Azure IoT Central. В этом кратком руководстве объясняется, как выполнять следующие задачи:

- Создание приложения Azure IoT Central, которое содержит пример _шаблона устройства_ и _имитированные устройства_.
- Просмотр возможностей шаблона устройства **Охлаждаемый торговый автомат** в приложении.
- Просмотр данных телеметрии и аналитических сведений из имитированного **охлаждающего** устройства.

В рамках этого краткого руководства вы просмотрите сведения имитированного **охлаждающего** устройства с помощью шаблона устройства. Имитированное устройство выполняет следующие операции:

* Отправляет в приложение данные телеметрии, такие как температура и давление.
* Отправляет в приложение сообщения о значении свойств устройства, например оповещение о перемещении.
* Содержит параметры устройств, которые можно задать в приложении, например скорость вращения вентилятора.

Создание имитируемого устройства с помощью шаблона устройства в приложении Azure IoT Central позволяет проверить приложение перед подключением реального устройства.

## <a name="create-the-application"></a>Создание приложения

Для выполнения инструкций в этом кратком руководстве необходимо создать приложение Azure IoT Central с помощью шаблона приложения **Образец Contoso**.

Перейдите на страницу [диспетчера приложений](https://aka.ms/iotcentral) в Azure IoT Central. Затем введите адрес электронной почты и пароль, которые вы используете для доступа к подписке Azure:

![Ввод учетной записи организации](media/quick-deploy-iot-central/sign-in.png)

Чтобы начать создание нового приложения Azure IoT Central, выберите **Новое приложение**:

![Страница диспетчера приложений Azure IoT Central](media/quick-deploy-iot-central/iotcentralhome.png)

Чтобы создать приложение Azure IoT Central, сделайте следующее:

1. Выберите план оплаты **Free Trial Application** (Бесплатная пробная версия приложения).
1. Выберите понятное имя для приложения, например **Contoso IoT**. Azure IoT Central создаст уникальный префикс URL-адреса. Вы можете изменить этот префикс на что-то более запоминающееся.
1. Выберите шаблон приложения **Sample Contoso** (Образец Contoso).
1. Щелкните **Создать**.

![Страница создания приложения Azure IoT Central](media/quick-deploy-iot-central/iotcentralcreate.png)

## <a name="navigate-to-the-application"></a>Переход к приложению

Когда приложение будет готово, откроется его **домашняя страница**. В правом верхнем углу страницы можно включить _режим конструктора_, чтобы изменить домашнюю страницу. URL-адресом приложения является указанный ранее URL-адрес:

![Страница конструктора приложений](media/quick-deploy-iot-central/apphome.png)

_Меню навигации слева_ позволяет получать доступ к различным разделам нового приложения Azure IoT Central:

![Меню навигации слева](media/quick-deploy-iot-central/navbar.png)

Чтобы просмотреть шаблоны устройств и устройства в приложении, выберите **Device Explorer** в меню навигации слева. Пример приложения содержит шаблон устройства **Охлаждаемый торговый автомат**. На основе этого шаблона устройства уже создано три имитированных устройства:

![Обозреватель устройств](media/quick-deploy-iot-central/deviceexplorer.png)

## <a name="view-the-device-template-and-devices"></a>Просмотр шаблона устройства и устройств

Чтобы просмотреть охлаждающее устройство, созданное с помощью шаблона устройства **Охлаждаемый торговый автомат**, выполните следующие действия. Шаблон устройства определяет следующее:

* _Измерения_, отправляемые с устройства, например данные телеметрии температуры.
* _Параметры_, позволяющие управлять устройством, например скорость вращения вентилятора.
* _Свойства_, хранящие сведения об устройстве, например серийный номер.
* [Правила](howto-create-telemetry-rules.md), позволяющие автоматизировать действия на основе поведения устройства.
* Настраиваемую _панель мониторинга_, отображающую сведения об устройстве.

На основе шаблона устройства можно создать имитированное и реальное устройство.

### <a name="measurements"></a>Измерения

Для **охлаждающего устройства 1** отображается страница **Measurements** (Измерения). Вы можете просмотреть список измерений, отправленных с имитированного устройства. На странице также отображается настраиваемая диаграмма видимых измерений:

![Страница измерений](media/quick-deploy-iot-central/measurements.png)

Вы можете переключать видимость отдельных элементов и настраивать диаграмму. Текущая диаграмма содержит данные телеметрии из имитированного устройства. При наличии соответствующих разрешений для шаблона устройства можно добавить новые измерения.

> [!NOTE]
> Может потребоваться некоторое время, прежде чем имитированные данные появятся на диаграмме.

### <a name="settings"></a>Параметры

Выберите **Параметры**. На странице **параметров** можно управлять устройством. Например, вы можете обновить скорость вращения вентилятора в охлаждающем устройстве:

![Параметры](media/quick-deploy-iot-central/settings.png)

Если устройство подтвердит изменение, параметр отобразится как **синхронизированный**.

### <a name="properties"></a>properties

Выберите **Свойства**. На странице **свойств** можно выполнять следующие действия:

* Сохранить сведения об устройстве, например имя клиента.
* Просматривать значения свойств, сообщаемые устройством, например оповещение о перемещении.

![properties](media/quick-deploy-iot-central/properties.png)

### <a name="dashboard"></a>панель мониторинга

Выберите **Панель мониторинга**. Панель мониторинга — это настраиваемое представление сведений об устройстве, например измерений, свойств и ключевых показателей эффективности:

![панель мониторинга](media/quick-deploy-iot-central/dashboard.png)

## <a name="view-analytics"></a>Просмотр аналитических сведений

В предыдущем разделе показано, как просматривать сведения об отдельном устройстве. С помощью [наборов устройств](howto-use-device-sets.md) и [аналитики](howto-create-analytics.md) можно просмотреть сводные сведения о нескольких устройствах.

Набор устройств с помощью запроса динамически выбирает группы устройств, отвечающие условиям. Например, набор устройств **Machines in Seattle** (Устройства в Сиэтле) выбирает охлаждающие устройства, расположенные в Сиэтле. Чтобы просмотреть набор устройств **Machines in Seattle** (Устройства в Сиэтле), в меню навигации слева выберите **Device Sets** (Наборы устройств), а затем — **Machines in Seattle** (Устройства в Сиэтле):

![Набор устройств "Устройства в Сиэтле"](media/quick-deploy-iot-central/deviceset.png)

Вы можете просматривать аналитические данные для устройств в наборе на странице **Аналитика**:

![Аналитические сведения об устройствах в Сиэтле](media/quick-deploy-iot-central/analytics.png)
