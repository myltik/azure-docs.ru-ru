# <a name="what-are-app-passwords-in-azure-multi-factor-authentication"></a>Что такое пароли приложений в службе Многофакторной идентификации Azure
Сейчас некоторые не использующие браузер приложения, например собственный почтовый клиент Apple, использующий Exchange Active Sync, не поддерживают многофакторную проверку подлинности. Многофакторная проверка подлинности активируется для каждого пользователя.  Это означает, что пользователь не может использовать многофакторную проверку подлинности в таких случаях:

- Пользователь включил многофакторную проверку подлинности.
- Пользователь пытается использовать внебраузерные приложения.

Пароль приложения дает пользователю возможность использовать приложение.

Получив пароль, используйте его вместо исходного пароля в этих приложениях без использования браузера. При настроенной двухфакторной проверке подлинности корпорация Майкрософт не позволит войти с вашим паролем программе, которая не может пройти второй этап проверки. Встроенный почтовый клиент Apple не умеет выполнять двухфакторную проверку подлинности, поэтому он не сможет войти от вашего имени. Чтобы решить эту проблему, нужно создать более надежный пароль приложения, который не используется постоянно. Пароли приложений предназначены только для тех приложений, которые не поддерживают двухфакторную проверку подлинности. Используйте пароль приложения, чтобы приложения могли обойти многофакторную идентификацию и продолжить работу.


> [!NOTE]
> Все клиенты Office 2013 (включая Outlook) поддерживают новые протоколы идентификации и двухфакторную проверку подлинности. Пароли приложений не нужно использовать с клиентами Office 2013.  Дополнительные сведения см. в записи блога [Office 2013 modern authentication public preview announced](https://blogs.office.com/2015/03/23/office-2013-modern-authentication-public-preview-announced/) (Анонсирована общедоступная предварительная версия современной службы многофакторной идентификации для Office 2013).


## <a name="how-to-use-app-passwords"></a>Использование паролей приложений
Ниже представлена важная информация о паролях приложений:

* Вам не нужно придумывать пароли приложений. Они создаются автоматически.
* В настоящее время имеется ограничение — не более 40 паролей на пользователя.
* Если вы попытаетесь создать пароль после того, как создано максимальное количество паролей, вам потребуется удалить один из имеющихся паролей приложений, чтобы создать новый.
* Мы рекомендуем использовать один пароль приложения для каждого устройства, а не для отдельных приложений. Например, вы можете создать один пароль приложения для ноутбука и использовать его для всех приложений на ноутбуке. Затем создайте второй пароль приложения для всех приложений на рабочем столе.
* При настройке двухфакторной проверки подлинности вам сразу предоставляется один пароль приложения.  Если потребуются дополнительные пароли, их можно создать.



## <a name="creating-and-deleting-app-passwords"></a>Создание и удаление паролей приложений
При первом входе вам предоставляется пароль приложения.  Кроме того, позже вы можете создавать и удалять дополнительные пароли приложений. Как вы это будете делать, зависит от способа использования многофакторной проверки подлинности. Следующие вопросы помогут вам определить, где можно управлять паролями приложений.

1. Вы используете двухфакторную проверку подлинности для личной учетной записи Майкрософт? Если да, см. статью о [паролях приложений и двухфакторной проверке подлинности](https://support.microsoft.com/help/12409/microsoft-account-app-passwords-two-step-verification). Если нет, переходите ко второму вопросу.

2. Итак, вы используете двухфакторную проверку подлинности для рабочей или учебной учетной записи. Выполняете ли вы вход в приложения Office 365? Если да, см. статью [Создание пароля приложения для Office 365](https://support.office.com/article/Create-an-app-password-for-Office-365-3e7c860f-bda4-4441-a618-b53953ee1183). Если нет, переходите к третьему вопросу.

3. Вы используете двухфакторную проверку подлинности для Microsoft Azure? Если да, переходите к разделу [Управление паролями приложений на портале Azure](#manage-app-passwords-in-the-Azure-portal) в этой статье. Если нет, переходите к четвертому вопросу.

4. Вы не знаете, где вы используете двухфакторную проверку подлинности? Если да, переходите к разделу [Управление паролями приложения на портале MyApps](#manage-app-passwords-with-the-myapps-portal) в этой статье.


## <a name="manage-app-passwords-in-the-azure-portal"></a>Управление паролями приложений на портале Azure
Если вы используете многофакторную проверку подлинности для Azure, пароли следует создавать на портале Azure.

### <a name="to-create-app-passwords-in-the-azure-portal"></a>Создание паролей приложений на портале Azure
1. Войдите на портал Azure.
2. В верхней части страницы щелкните свое имя пользователя и выберите **Изменить пароль**.
3. На странице подтверждения вверху выберите **Пароли приложений**.
4. Нажмите кнопку **Создать**.
5. Введите имя для пароля приложения и щелкните **Далее**.
6. Скопируйте пароль приложения в буфер обмена и вставьте его в свое приложение.

   ![Облако](./media/multi-factor-authentication-end-user-app-passwords/app2.png)


## <a name="manage-app-passwords-with-the-myapps-portal"></a>Управление паролями приложения на портале MyApps.
Даже если вы не знаете, где вы используете многофакторную идентификацию, пароли приложений можно всегда создавать или удалять на портале Myapps.

### <a name="to-create-an-app-password-using-the-myapps-portal"></a>Создание пароля приложения с помощью портала MyApps
1. Войдите на портал по адресу [https://myapps.microsoft.com](https://myapps.microsoft.com).
2. Щелкните свое имя в правом верхнем углу и выберите **Профиль**.
3. Щелкните **Дополнительная проверка безопасности**.
   ![Выбор дополнительной проверки безопасности — снимок экрана](./media/multi-factor-authentication-end-user-manage/myapps1.png)

4. Выберите **Пароли приложений**.
   ![Выбор паролей приложений — снимок экрана](./media/multi-factor-authentication-end-user-app-passwords/apppass2.png)

5. Нажмите кнопку **Создать**.
6. Введите имя для пароля приложения и нажмите кнопку **Далее**.
7. Скопируйте пароль приложения в буфер обмена и вставьте его в свое приложение.
   ![Создание пароля приложения](./media/multi-factor-authentication-end-user-app-passwords/create2.png)

### <a name="to-delete-an-app-password-using-the-myapps-portal"></a>Удаление пароля приложения с помощью портала MyApps
1. Войдите на портал по адресу [https://myapps.microsoft.com](https://myapps.microsoft.com).
2. Вверху выберите профиль.
3. Щелкните **Дополнительная проверка безопасности**.

   ![Выбор дополнительной проверки безопасности — снимок экрана](./media/multi-factor-authentication-end-user-manage/myapps1.png)

4. Выберите **Пароли приложений**.

   ![Выбор паролей приложений — снимок экрана](./media/multi-factor-authentication-end-user-app-passwords/apppass2.png)

5. Рядом с паролем приложения, который следует удалить, щелкните **Удалить**.

   ![Удаление пароля приложения](./media/multi-factor-authentication-end-user-app-passwords/delete1.png)

6. Чтобы подтвердить удаление пароля, нажмите кнопку **Да**.
7. Когда пароль будет удален, нажмите кнопку **Закрыть**.
