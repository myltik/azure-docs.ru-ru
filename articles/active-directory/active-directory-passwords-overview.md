# <a name="azure-ad-self-service-password-reset-for-the-it-professional"></a>Самостоятельный сброс пароля в Azure AD для ИТ-специалистов

"Самообслуживание" — это модное слово внутри многих ИТ-отделов по всему миру, имеющее различные значения. Сегодня на рынке предлагается широкий выбор продуктов, позволяющих управлять локальными группами, паролями и профилями пользователей в облаке и локально.

Самостоятельный сброс пароля (SSPR) в Azure Active Directory (Azure AD) выделяется на общем фоне благодаря своей простоте и легкости развертывания. Самостоятельный сброс пароля в Azure AD объединяет набор возможностей, которые:

* позволяют пользователям управлять паролями:
  * на любом устройстве;
  * в любом месте;
  * в любое время;
* поддерживают соответствия политикам, которые вы определили от имени администратора.

Если вы готовы, вы можете приступить к работе с Azure AD SSPR, используя [краткое руководство по началу работы](active-directory-passwords-getting-started.md), и быстро реализовать возможность сброса паролей для пользователей.

## <a name="what-is-possible"></a>Возможности

* **Самостоятельное изменение пароля** позволяет пользователям или администраторам менять свои пароли без помощи администратора.
* **Самостоятельная разблокировка учетной записи** позволяет пользователям разблокировать учетную запись без помощи администратора.
* **Самостоятельный сброс пароля** позволяет пользователям или администраторам автоматически сбрасывать свои пароли без помощи администратора. Для самостоятельного сброса пароля требуется Azure AD Premium или Basic. [Выпуски Azure Active Directory.](active-directory-editions.md)
* **Сброс пароля администратором** позволяет администратору сбросить пароль пользователя или другого администратора на [портале Azure](https://docs.microsoft.com/azure/azure-portal-overview).
* **Отчеты по операциям управления паролями** позволяют администраторам лучше понять ситуацию с регистрацией и сбросом паролей в организации. [Reporting options for Azure AD password management](active-directory-passwords-reporting.md) (Параметры отчетов для управления паролями Azure AD).
* **Обратная запись паролей** позволяет управлять локальными паролями из облака, чтобы федеративный пользователь или пользователь с синхронизированным паролем смог выполнять все предыдущие сценарии или эти сценарии можно было выполнить от его лица. Для обратной записи паролей требуется выпуск [Azure AD Premium](active-directory-get-started-premium.md).

## <a name="why-choose-azure-ad-self-service-password-reset"></a>В чем преимущества самостоятельного сброса пароля в Azure AD

* **Снижение затрат.** На техническую поддержку и сброс паролей обычно приходится 20 % расходов ИТ-организации.
* **Повышение удобства работы пользователей** и **сокращение нагрузки на службу технической поддержки** благодаря предоставлению пользователям возможности сразу устранять собственные проблемы с паролями, не обращаясь в службу технической поддержки.
* **Поддержка мобильности.** Пользователи могут сбрасывать пароли независимо от своего местонахождения.

## <a name="azure-ad-self-service-password-reset-availability"></a>Доступность самостоятельного сброса пароля в Azure AD

В зависимости от подписки самостоятельный сброс пароля в Azure AD доступен на трех уровнях.

* **Бесплатная подписка Azure AD.** Администраторы облака могут сбрасывать свои пароли.
* **Azure AD Basic** или любая **платная подписка Office 365.** Пользователи и администраторы облака могут сбрасывать свои пароли.
* **Azure AD Premium.** Любой пользователь или администратор, включая пользователей облака, федеративных пользователей и пользователей с синхронизацией паролей, могут сбрасывать свои пароли. Для локальных паролей требуется включить компонент обратной записи паролей.

## <a name="azure-ad-self-service-password-reset-a-sum-of-the-parts"></a>Компоненты самостоятельного сброса пароля в Azure AD

Самостоятельный сброс пароля в Azure AD состоит из следующих компонентов:

* **Портал настройки управления паролями.** Здесь можно контролировать параметры управления паролями в своих клиентах с помощью портала Azure.
* **Портал регистрации данных для сброса паролей.** На этом портале пользователи могут самостоятельно зарегистрировать необходимые данные для сброса паролей.
* **Портал сброса паролей.** На этом портале пользователи могут сбросить свои пароли, пройдя заданное администратором количество проверок и дав ответы на соответствующие вопросы.
* **Портал изменения пользовательских паролей.** Пользователи могут изменять свои пароли, введя старый пароль и выбрав новый.
* **Отчеты об управлении паролями.** Администраторы могут просматривать и анализировать отчеты о действиях с паролями клиентов на портале Azure.
* **Компонент обратной записи паролей в локальной службе с использованием Azure AD Connect.** Позволяет управлять локальными, федеративными пользователями, а также пользователями с синхронизацией паролей из облака.
