# <a name="post-configuration-tasks-for-hybrid-azure-ad-join"></a>Задачи постконфигурации для гибридного присоединения устройств к Azure AD

После выполнения Azure AD Connect для настройки в вашей организации гибридного присоединения устройств к Azure AD существуют несколько дополнительных шагов, которые необходимо выполнить для завершения установки.  Выполните только шаги, которые применяются для ваших устройств.

## <a name="1-configure-controlled-rollout-optional"></a>1. Настройка контролируемого выпуска (необязательно)
Все присоединенные к домену устройства под управлением Windows 10 и Windows Server 2016 будут выполнять автоматическую регистрацию в Azure AD после завершения всех шагов по настройке. Если вы предпочитаете контролируемый выпуск, а не автоматическую регистрацию, вы можете использовать групповую политику для выборочного включения или отключения автоматического выпуска.  Эту групповую политику следует задать до выполнения других шагов по настройке в Azure AD.
* Создайте объект групповой политики в Active Directory.
* Назовите его (например, "Гибридное присоединение к Azure AD").
* Внесите изменения и выберите "Конфигурация компьютера > Политики > Административные шаблоны > Компоненты Windows > Регистрация устройств".

>[!NOTE]
>Настройки политики для версии 2012R2 находятся в следующем расположении: **Конфигурация компьютера > Политики > Административные шаблоны > Компоненты Windows > Присоединение к рабочей области > Automatically workplace join client computers (Автоматическое присоединение к рабочей области клиентских компьютеров)**.

* Отключите параметр: Register domain-joined computers as devices (Регистрация подключенных к домену компьютеров как устройств).
* Примените изменения и нажмите кнопку "ОК".
* Свяжите объект групповой политики (GPO) с выбранным расположением (подразделения, группы безопасности или домена для всех устройств).

## <a name="2-configure-network-with-device-registration-endpoints"></a>2. Настройка сети с конечными точками регистрации устройств
Убедитесь, что следующие URL-адреса доступны с компьютеров в сети организации для регистрации в Azure AD:

* https://enterpriseregistration.windows.net
* https://login.microsoftonline.com
* https://device.login.microsoftonline.com 

## <a name="3-implement-wpad-for-windows-10-devices"></a>3. Реализация WPAD для устройств Windows 10
Если организация обращается к Интернету через исходящий прокси-сервер, реализуйте автоматическое обнаружение веб-прокси (WPAD), чтобы компьютеры Windows 10 можно было зарегистрировать в Azure AD.

## <a name="4-configure-the-scp-in-any-forests-that-were-not-configured-by-azure-ad-connect"></a>4. Настройка точки подключения службы (SCP) в любом лесе, который не был настроен с помощью Azure AD Connect 

Точка подключения службы (SCP) содержит сведения о клиенте Azure AD, которые будут использоваться устройствами для автоматической регистрации.  Запустите скрипт PowerShell, ConfigureSCP.ps1, загруженный из Azure AD Connect.

## <a name="5-configure-any-federation-service-that-was-not-configured-by-azure-ad-connect"></a>5. Конфигурация ненастроенной для Azure AD Connect службы федерации

Если ваша организация использует службу федерации для входа в Azure AD, то правила утверждений в вашем отношении доверия с проверяющей стороной Azure AD должны разрешать проверку подлинности устройства. Если вы используете федерацию с AD FS, перейдите к [справке AD FS](https://aka.ms/aadrptclaimrules) для создания правил утверждений. При использовании решения федерации стороннего поставщика обратитесь к поставщику для получения рекомендаций.  

>[!NOTE]
>Если у вас есть устройства Windows нижнего уровня, служба должна поддерживать выдачу утверждений authenticationmethod и wiaormultiauthn при получении запросов на отношение доверия Azure AD. В AD FS следует добавить правило преобразования выдачи, пропускающее метод проверки подлинности.

## <a name="6-enable-azure-ad-seamless-sso-for-windows-down-level-devices"></a>6. Включение простого единого входа Azure AD для устройств Windows нижнего уровня

Если в организации используется синхронизация хэшей паролей или сквозная проверка подлинности для входа в Azure AD, включите простой единый вход Azure AD с этим методом входа для проверки подлинности устройств Windows нижнего уровня: https://docs.microsoft.com/en-us/azure/active-directory/connect/active-directory-aadconnect-sso. 

## <a name="7-set-azure-ad-policy-for-windows-down-level-devices"></a>7. Настройка политики Azure AD для устройств Windows нижнего уровня

Чтобы поддерживать регистрацию устройств Windows нижнего уровня, необходимо убедиться, что политика Azure AD разрешает пользователям регистрировать устройства. 

* Войдите в учетную запись на портале Azure.
* Выберите "Azure Active Directory > Устройства > Параметры устройств".
* Для параметра "Пользователи могут регистрировать устройства в Azure AD" выберите значение "Все".
* Щелкните Сохранить

## <a name="8-add-azure-ad-endpoint-to-windows-down-level-devices"></a>8. Добавление конечной точки Azure AD для устройств Windows нижнего уровня

В локальные зоны интрасети добавьте конечную точку проверки подлинности устройств в Azure AD на устройствах Windows нижнего уровня, чтобы избежать запросов сертификатов при проверке подлинности устройств: https://device.login.microsoftonline.com 

Если вы используете [простой единый вход](https://aka.ms/hybrid/sso), а также включили в этой зоне параметр "Разрешить обновление строки состояния в сценарии", то добавьте следующую конечную точку: https://autologon.microsoftazuread-sso.com 

## <a name="9-install-microsoft-workplace-join-on-windows-down-level-devices"></a>9. Установка Microsoft Workplace Join на устройствах Windows нижнего уровня

Установщик создает в системе устройства запланированную задачу, которая выполняется в контексте пользователя. Задача запускается в момент входа пользователя в систему Windows. Эта задача автоматически присоединяет устройство к Azure AD, используя учетные данные пользователя, после проверки подлинности с использованием встроенной проверки подлинности Windows. Центр скачивания находится по ссылке https://www.microsoft.com/en-us/download/details.aspx?id=53554. 

## <a name="10-configure-group-policy-to-allow-device-registration"></a>10. Настройка групповой политики для разрешения регистрации устройств

* Создайте объект групповой политики в Active Directory, если он еще не создан.
* Назовите его (например, "Гибридное присоединение к Azure AD").
* внесите изменения и выберите "Конфигурация компьютера > Политики > Административные шаблоны > Компоненты Windows > Регистрация устройств".
* Включите параметр Register domain-joined computers as devices (Регистрация подключенных к домену компьютеров как устройств).
* Примените изменения и нажмите кнопку "ОК".
* Свяжите объект групповой политики (GPO) с выбранным расположением (подразделения, группы безопасности или домена для всех устройств).

>[!NOTE]
>Настройки политики для версии 2012R2 находятся в следующем расположении: **Конфигурация компьютера > Политики > Административные шаблоны > Компоненты Windows > Присоединение к рабочей области > Automatically workplace join client computers (Автоматическое присоединение к рабочей области клиентских компьютеров)**.
