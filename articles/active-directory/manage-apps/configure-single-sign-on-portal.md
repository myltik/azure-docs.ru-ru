# <a name="managing-single-sign-on-for-enterprise-apps"></a>Управление параметрами единого входа для корпоративных приложений

В этой статье описывается управление параметрами единого входа для корпоративных приложений с помощью [портала Azure](https://portal.azure.com). Корпоративные приложения — это приложения, развертываемые и используемые в вашей организации. Эта статья в частности касается приложений, которые были добавлены из [коллекции приложений Azure Active Directory](what-is-single-sign-on.md#get-started-with-the-azure-ad-application-gallery). 

## <a name="finding-your-apps-in-the-portal"></a>Поиск приложений на портале
Все корпоративные приложения, для которых был настроен единый вход, можно просматривать и администрировать на портале Azure. Приложения можно найти в разделе портала **Все службы** &gt; **Корпоративные приложения**. 

![колонка "Корпоративные приложения"](./media/configure-single-sign-on-portal/enterprise-apps-blade.png)

Выберите **Все приложения**, чтобы просмотреть список всех настроенных приложений. При выборе приложения отображается колонка ресурсов с отчетами для этого приложения и различными управляемыми параметрами.

Чтобы управлять параметрами единого входа, выберите **Единый вход**.

![Колонка ресурсов приложения](./media/configure-single-sign-on-portal/enterprise-apps-sso-blade.png)

## <a name="single-sign-on-modes"></a>Режимы единого входа
Колонка **Единый вход** начинается с меню **Режим**, в котором можно настроить режим единого входа. Доступны следующие параметры.

* **Вход на основе SAML**. Этот параметр доступен, если приложение поддерживает федеративный единый вход с помощью Azure Active Directory по протоколу SAML 2.0, WS-Federation или OpenID Connect.
* **Вход по паролю**. Этот параметр доступен, если Azure AD поддерживает заполняемую для приложения форму пароля.
* **Вход по ссылке** — ранее назывался "Существующий единый вход". Этот параметр позволяет администраторам помещать ссылку для перехода в приложение на пользовательскую панель доступа Azure AD или в средство запуска приложений Office 365.

Дополнительные сведения об этих режимах см. в разделе [Принцип работы единого входа с Azure Active Directory](what-is-single-sign-on.md#how-does-single-sign-on-with-azure-active-directory-work).

## <a name="saml-based-sign-on"></a>Вход на основе SAML
При выборе параметра **Вход на основе SAML** откроется колонка, которая состоит из четырех частей:

### <a name="domains-and-urls"></a>Домены и URL-адреса
В этом разделе приведены все сведения о доменах и URL-адресах приложения, добавленных в каталог Azure AD. Все параметры, которые необходимо указать для включения в приложении единого входа, отображаются непосредственно на экране. Чтобы просмотреть все дополнительные параметры, установите флажок **Показать дополнительные параметры URL-адресов**. Ниже приведен список поддерживаемых параметров.

* **URL-адрес входа**, по которому пользователь переходит для входа в это приложение. Если приложение настроено для выполнения единого входа, инициированного поставщиком услуг, то при переходе пользователя на этот URL-адрес поставщик услуг выполняет перенаправление в Azure AD для проверки подлинности и входа пользователя. 
  * Если это поле заполнено, Azure AD использует этот URL-адрес для запуска приложения из Office 365 и панели доступа Azure AD.
  * Если это поле оставить пустым, Azure AD выполняет единый вход, инициированный поставщиком удостоверений, при запуске приложения из Office 365, с панели доступа Azure AD или при переходе по URL-адресу единого входа Azure AD.
* **Идентификатор**. Этот универсальный код ресурса должен однозначно определять приложение, для которого выполняется настройка единого входа. Это значение, которое Azure AD отправляет назад в приложение в качестве параметра "Аудитория" токена SAML, а приложение должно его проверить. Это значение также отображается как идентификатор сущности во всех метаданных SAML, предоставленных приложением.
* **URL-адрес ответа**. URL-адрес ответа указывает место, где приложение ожидает токен SAML. Он также называется URL-адресом службы обработчика утверждений (ACS). После ввода адресов нажмите кнопку "Далее", чтобы перейти на следующий экран. Этот экран сообщает, какие параметры вам нужно настроить на стороне приложения, чтобы оно принимало токен SAML от Azure AD.
* **Состояние ретранслятора** — это необязательный параметр, используемый, чтобы указать приложению, куда необходимо перенаправить пользователя после завершения аутентификации. Обычно в качестве значения для этого параметра используется допустимый URL-адрес приложения, но в некоторых приложениях это поле заполняется другими значениями. (Дополнительные сведения см. в документации по единому входу для приложения.) Возможность задать состояние ретрансляции — это новая возможность, доступная на новом портале Azure.

### <a name="user-attributes"></a>Атрибуты пользователя
В этом разделе администраторы могут просмотреть и изменить атрибуты, передаваемые в токене SAML, выдаваемом Azure AD каждый раз при входе в приложение.

Единственным поддерживаемым редактируемым атрибутом является атрибут **Идентификатор пользователя**. Это поле в Azure AD, однозначно определяющее каждого пользователя в приложении. Например, если при развертывании приложения в качестве имени пользователя и уникального идентификатора используется значение адреса электронной почты, то в Azure AD значение необходимо задать в поле user.mail.

### <a name="saml-signing-certificate"></a>Сертификат для подписи токена SAML
В этом разделе приведены сведения о сертификатах, используемых Azure AD для подписи токенов SAML, выдаваемых приложению каждый раз, когда пользователь проходит проверку подлинности. Это позволяет проверять свойства имеющегося сертификата, в том числе дату окончания срока их действия.

### <a name="application-configuration"></a>Конфигурация приложения
В последнем разделе представлена документация и (или) элементы управления, необходимые для настройки Azure Active Directory в качестве поставщика удостоверений для приложения.

В раскрывающемся меню **Настройка приложения** содержатся краткие встроенные инструкции по настройке приложения. Это еще одна новая уникальная функция на новом портале Azure.

## <a name="password-based-sign-on"></a>Единый вход по паролю
Если этот параметр поддерживается приложением, чтобы включить его, выберите режим единого входа на основе пароля, а затем щелкните **Сохранить**. Дополнительные сведения о развертывании единого входа на основе пароля см. в разделе [Принцип работы единого входа с Azure Active Directory](what-is-single-sign-on.md#how-does-single-sign-on-with-azure-active-directory-work).

![Единый вход по паролю](./media/configure-single-sign-on-portal/enterprise-apps-blade-password-sso.png)

## <a name="linked-sign-on"></a>Связанный единый вход
Если этот параметр поддерживается приложением, чтобы добавить URL-адрес, на который будет перенаправляться пользователь после выбора ссылки на панели доступа Azure AD или в средстве запуска приложений Office 365, выберите режим связанного единого входа. Дополнительные сведения о развертывании единого входа по ссылке (ранее известного как "Существующий единый вход") см. в разделе [Принцип работы единого входа с Azure Active Directory](what-is-single-sign-on.md#how-does-single-sign-on-with-azure-active-directory-work).

![Связанный единый вход](./media/configure-single-sign-on-portal/enterprise-apps-blade-linked-sso.png)
