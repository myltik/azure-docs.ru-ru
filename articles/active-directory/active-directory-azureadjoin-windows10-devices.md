# <a name="using-windows-10-devices-in-your-workplace"></a>Устройства под управлением Windows 10 в вашей рабочей области
Область применения: ПК под управлением Windows 10

Windows 10 предлагает организациям три модели безопасного и удобного доступа пользователей к рабочим ресурсам.

* **Присоединение к Azure Active Directory** (присоединение к Azure AD) — для сотрудников, которые используют преимущественно облачные ресурсы, например Office 365. Присоединение к Azure AD — это функция для самостоятельной подготовки рабочей среды, которая появилась в Windows 10.
* **Присоединение к домену**— для организаций, которые вкладывают средства в локальные приложения и ресурсы. В случае присоединения к домену работу в Windows 10 можно сделать более удобной, подключившись к Azure AD.
* **Новая упрощенная процедура "принеси свое устройство" (BYOD)**— для пользователей, которые хотят добавить в Windows рабочую или учебную учетную запись, обеспечивая простой доступ к ресурсам с личных устройств.

Следующая таблица содержит краткий обзор возможностей Windows 10, предоставляемых пользователям и ИТ-администраторам в рамках разных сценариев подготовки и использования устройств на предприятии.

| --- | --- | --- | --- |
| Вход с устройства Windows для рабочих или учебных учетных записей. |Да |Да |Нет |
| Единый вход пользователей для приложений Office 365 и Azure AD. Единый вход — это возможность выполнить вход только один раз для доступа ко всем ресурсам организации. |Да |Да |Да |
| Единый вход пользователей в приложения Kerberos или NTLM. |Да |Ограничено |Да, через VPN |
| Строгая проверка подлинности и удобный вход для рабочих или учебных учетных записей с помощью Microsoft Passport и Windows Hello. |Да |Да |Да |
| Доступ к Магазину Windows в организации с помощью рабочих или учебных учетных записей (а не учетной записи Майкрософт). |Да |Да |Да |

## <a name="use-work-owned-devices-with-azure-ad-join-and-domain-join-in-windows-10"></a>Использование корпоративных устройств для присоединения к Azure AD, а также присоединения к домену в Windows 10
Windows 10 предлагает два способа доступа к рабочим ресурсам с корпоративных устройств:

* Присоединение к Azure AD
* Присоединение к домену
  
  Оба варианта имеют свои преимущества в зависимости от нужд и требований организации. В некоторых случаях будет полезно сочетать оба метода развертывания.

## <a name="when-to-use-azure-active-directory-join"></a>Когда лучше использовать присоединение к Azure Active Directory
Присоединение к Azure AD — это новая функция для самостоятельной подготовки рабочей среды, которая появилась в Windows 10.  Она предназначена для тех сотрудников, которые используют преимущественно облачные ресурсы, например Office 365. Это простой способ настроить компьютер, планшет или телефон для рабочих нужд. Устройства управляются через MDM. Для всех платформ Windows вы получите одинаковые элементы управления.

**Присоединение к Azure AD рекомендуется в следующих случаях**:

* вам нужна поддержка самостоятельной подготовки устройств для мобильных сотрудников;
* вы предоставляете пользователям корпоративные мобильные устройства, например планшеты или телефоны;
* вы хотите перенести из Active Directory в Azure AD часть пользователей, например сезонных работников, подрядчиков или учащихся;
* вы хотите, чтобы работники удаленных филиалов с ограниченной локальной инфраструктурой могли подключаться к основной сети;
* у вас нет локальной службы каталогов Active Directory.

В некоторых организациях присоединение к Azure AD будет основным способом развертывания корпоративных устройств. Такой сценарий наиболее актуален при миграции в облако всех ресурсов организации или их большей части. Гибридные организации, которые используют как локальную службу каталогов Active Directory, так и Azure AD, могут выбирать разные методы развертывания для разных пользователей или отделов.

Например, школы и университеты могут управлять учетными записями сотрудников через локальную службу каталогов AD, а учетными записями учащихся — через Azure AD. Некоторые компании предпочитают хранить в Azure AD учетные записи сотрудников удаленных офисов или отдела продаж. В гибридной организации могут сочетаться методы присоединения к Azure AD и присоединения к домену. Присоединение к Azure AD отлично сочетается с присоединением к домену в рамках развертывания устройств в рабочей среде.

**Если большинство пользователей используют корпоративные ресурсы через облако, организация получает дополнительные преимущества**:

* вы сможете устранить зависимости для локальной инфраструктуры удостоверений;
* вы сможете упростить модель развертывания устройств, перейдя от создания образов к самостоятельной конфигурации;
* вы сможете использовать MDM для управления всеми устройствами на различных платформах.

Дополнительную информацию о присоединении к Azure AD см. в статье [Использование возможностей облачных служб на устройствах с Windows 10 с помощью присоединения к Azure Active Directory](active-directory-azureadjoin-overview.md).

## <a name="when-to-use-domain-join-or-keep-using-it"></a>Когда лучше использовать (или продолжать использовать) присоединение к домену
Многие организации уже более 15 лет используют присоединение к домену для подключения рабочих устройств. Оно позволяет пользователям входить на устройства с помощью рабочих или учебных учетных записей Active Directory. Присоединение к домену также позволяет ИТ-персоналу централизованно и всесторонне управлять этими устройствами. Обычно организации создают для подготовки устройств системные образы, а для управления устройствами часто применяют System Center Configuration Manager (SCCM) или групповые политики.

**Присоединение к домену (или его дальнейшее использование) рекомендуется в следующих случаях**:

* У вас есть развернутые на мобильных устройствах приложения Win32, которые используют NTLM или Kerberos.
* Для управления устройствами вам нужны групповые политики или SCCM/DCM.
* Вы хотите и дальше использовать системные образы для настройки устройств сотрудников.

**Кроме того, присоединение к домену в Windows 10 обеспечивает следующие преимущества при подключении к Azure AD**:

* строгая проверка подлинности с привязкой к устройствам и удобный вход для рабочих или учебных учетных записей с помощью Microsoft Passport и Windows Hello;
* доступ к Магазину Windows в организации для устройств, на которых используются рабочие или учебные учетные записи (учетная запись Майкрософт не требуется);
* перенос пользовательских параметров между устройствами, которые используют рабочие или учебные учетные записи, с соблюдением корпоративных требований (учетная запись Майкрософт не требуется);
* Возможность ограничить доступ к корпоративным приложениям, предоставив его только устройствам, которые соответствуют корпоративным политикам.

Дополнительную информацию о присоединении к Azure AD см. в статье [Использование возможностей облачных служб на устройствах с Windows 10 с помощью присоединения к Azure Active Directory](active-directory-azureadjoin-overview.md).

## <a name="enable-joining-of-personally-owned-devices-for-work-or-school"></a>Поддержка присоединения собственных устройств пользователей для использования рабочих или учебных учетных записей
В рамках корпоративной поддержки BYOD Windows 10 позволяет пользователям добавлять рабочие или учебные учетные записи на свои компьютеры, планшеты и телефоны. Когда пользователь добавляет рабочую или учебную учетную запись, устройство регистрируется в Azure AD, а при необходимости — также в системе управления мобильными устройствами, если она настроена в организации. В каталоге такие устройства будут обозначены как зарегистрированные или присоединенные к Azure AD. ИТ-администраторы могут использовать эту информацию в различных политиках, например могут устанавливать для личных устройств менее строгий контроль, чем для корпоративных.

Пользователям очень удобно добавлять рабочие или учебные учетные записи на свои личные устройства. Это можно сделать при первом обращении к рабочему приложению либо вручную через меню "Параметры". Такая учетная запись обеспечит единый вход для доступа к ресурсам организации.

Дополнительную информацию о подключении к Azure AD см. в статье [Подключение присоединенных к домену устройств к Azure AD для работы в Windows 10](active-directory-azureadjoin-devices-group-policy.md).

## <a name="enable-domain-join-or-azure-ad-join"></a>Включение присоединения к домену или присоединения к Azure AD
Все методы, описанные выше (присоединение к домену, присоединение к Azure AD и добавление рабочей или учебной учетной записи), обеспечивают взаимодействие с пользователем Windows 10. Но чтобы пользователь мог воспользоваться любым из этих методов, ИТ-администратор должен включить поддержку этих функций в инфраструктуре.

## <a name="requirements-for-deploying-azure-ad-join"></a>Требования для развертывания присоединения к Azure AD
Чтобы развернуть присоединение к Azure AD, вам нужны:

* подписка Azure AD;
* подписка Azure AD Premium (если вам нужны дополнительные функции, например автоматическая регистрация в MDM).
* управление мобильными устройствами, например подписка Microsoft Intune, управление мобильными устройствами для Office 365 или любое партнерское решение для управления мобильными устройствами, которое интегрируется с Azure AD. (Дополнительные сведения см. в разделе [Часто задаваемые вопросы](#frequently-asked-questions) в конце этой статьи).

В гибридной среде настоятельно рекомендуется развернуть Azure AD Connect для расширения локального каталога в Azure AD.

## <a name="requirements-for-using-domain-join-with-azure-ad"></a>Требования для использования присоединения к домену с Azure AD
Присоединение к домену доступно, как и раньше. Но для расширения его возможностей до уровня Azure AD вам потребуется:

* подписка Azure AD;
* развертывание Azure AD Connect для расширения локального каталога до Azure AD;
* политика, которая предоставляет присоединенным к домену устройствам условный доступ к Azure AD;
* политика, которая предоставляет доступ к присоединенным к домену устройствам, если вы хотите иметь возможность ограничить доступ для некоторых устройств;
* System Center Configuration Manager (предварительная техническая версия 1509), чтобы включить правила обязательной совместимости устройств (см. документацию TechNet и запись блога).

Дополнительную информацию о подключении к Azure AD см. в статье [Подключение присоединенных к домену устройств к Azure AD для работы в Windows 10](active-directory-azureadjoin-devices-group-policy.md).

## <a name="requirements-for-using-byod-and-add-a-work-or-school-account"></a>Требования для использования BYOD и добавления рабочей или учебной учетной записи
Чтобы использовать функцию "принеси свое устройство" (BYOD) с рабочими или учебными учетными записями, вам потребуется:

* подписка Azure AD;
* подписка Azure AD Premium (если вам нужны дополнительные функции, например автоматическая регистрация в MDM).

## <a name="requirements-for-using-microsoft-passport"></a>Требования для использования службы Microsoft Passport
Чтобы включить поддержку Microsoft Passport, вам потребуется:

* инфраструктура открытых ключей (PKI) для поддержки проверки подлинности на основе сертификатов с использованием Microsoft Passport;
* подписка Intune для поддержки аутентификации на основе сертификатов с использованием Microsoft Passport для рабочих или учебных учетных записей и присоединения к Azure AD;
* System Center Configuration Manager, предварительная техническая версия 1509 (см. документацию TechNet и запись блога), для поддержки проверки подлинности на основе сертификатов с использованием Microsoft Passport для присоединения к домену;
* политика, которая разрешает использование Microsoft Passport в организации.

Вместо PKI можно использовать Microsoft Passport на основе ключей. Для этого следует:

* развернуть контроллеры домена Windows Server 2016, предварительная рабочая версия 1 (не обязательно на функциональном уровне домена или леса, так как вполне достаточно нескольких контроллеров для каждого сайта Active Directory, чтобы обеспечить достаточную избыточность);
* задать политику, которая разрешает корпоративное использование Microsoft Passport.

Дополнительные сведения о Microsoft Passport и Windows Hello в Windows 10 см. в статье <link-to-MS-Passport-and-Windows-Hello-document>.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы
### <a name="which-partner-mobile-device-management-products-integrate-with-azure-ad"></a>Какие партнерские продукты для управления мобильными устройствами интегрируются с Azure AD?
Для поддержки единой регистрации и условного доступа в Windows 10 можно интегрировать с Azure AD следующие продукты:

* AirWatch компании VMware
* Citrix Xenmobile
* Lightspeed Mobile Manager
* локальное управление мобильными устройствами SOTI.
* MobileIron

### <a name="what-about-workplace-join-in-windows-10"></a>Почему в Windows 10 нет присоединения к рабочей области?
Присоединение к рабочей области использовалось в Windows 8.1 для реализации функции BYOD. В Windows 10 функция BYOD реализована через добавление рабочей или учебной учетной записи, как описано выше в этом документе. В организациях, которые не интегрируют управление мобильными устройствами с Azure AD, пользователи могут регистрировать устройства в системе управления вручную, открыв **Параметры** > **Учетные записи** > **Рабочий доступ**.

### <a name="can-users-connect-their-microsoft-account-to-their-domain-account-in-windows-10"></a>Можно ли в Windows 10 подключить учетную запись Майкрософт к учетной записи домена?
В Windows 10 это невозможно. В Windows 8.1 пользователи устройств, подключенных к домену, могли "подключать" свои учетные записи Майкрософт (например, Hotmail, Live, Outlook, Xbox и т. д.) к учетной записи домена. Это позволяло выполнять единый вход в службы Live, использовать Магазин Windows и переносить параметры пользователя между устройствами. В Windows 10 функция "подключения" учетной записи Майкрософт удалена как устаревшая. Чтобы включить единый вход в клиентские службы, например в Магазин Windows, пользователь теперь может добавить одну или несколько учетных записей Майкрософт в качестве дополнительных. Для этого используйте раздел **Параметры** > **Учетные записи** > **Ваша учетная запись**.

Если пользователь обновляет Windows 8.1 до Windows 10 на подключенном к домену устройстве с подключенными учетными записями Майкрософт, все эти учетные записи будут автоматически добавлены в список дополнительных учетных записей.
