# <a name="sign-in-activity-reports-in-the-azure-active-directory-portal"></a>Отчеты о действиях входа на портале Azure Active Directory

Функция отчетов в Azure Active Directory (Azure AD) [на портале Azure](https://portal.azure.com) позволяет получать всю необходимую информацию, чтобы определить, как работает среда.

Архитектура создания отчетов в Azure Active Directory состоит из следующих компонентов.

- **Действие** 
    - **Действия входа** — информация об использовании управляемых приложений и действиях входа.
    - **Журналы аудита** — данные системных операций об управлении пользователями и группами, об управляемых приложениях и действиях с каталогами.
- **Безопасность** 
    - **Входы, представляющие риск**. Вход, представляющий риск, означает, что в систему пытался войти пользователь, который не является законным владельцем учетной записи. Дополнительные сведения см. в разделе "Вход, представляющий риск".
    - **Пользователи, находящиеся в группе риска**. Такая пометка означает, что конфиденциальность учетной записи пользователя, возможно, нарушена. Дополнительные сведения см. в разделе "Пользователи, помеченные для события риска".

В этом разделе содержатся общие сведения о действиях входа.

## <a name="pre-requisite"></a>Предварительные требования

### <a name="who-can-access-the-data"></a>Кто может получить доступ к данным?
* Пользователи с ролью администратора безопасности или читателя безопасности
* Глобальные администраторы
* Любой пользователь (не администратор) может получить доступ к своим данным о действиях входа. 

### <a name="what-azure-ad-license-do-you-need-to-access-sign-in-activity"></a>Какие лицензии Azure AD требуются для доступа к действию входа?
* Для просмотра отчета о всех действиях входа с клиентом должна быть связана лицензия Azure AD Premium.


## <a name="signs-in-activities"></a>Действия входа

Информация, доступная в отчете о входе пользователя, поможет вам ответить на такие вопросы:

* Что такое шаблон входа пользователя?
* Сколько пользователей входили в течение недели?
* Каков статус их входа?

Знакомство с данными о действиях входа следует начать с раздела **События входа** в разделе "Действие" службы **Azure Active**.


![Действие входа](./media/active-directory-reporting-activity-sign-ins/61.png "Действие входа")


Журнал аудита содержит представление списка по умолчанию, в котором отображаются:

- пользователь;
- приложение, в которое вошел пользователь;
- состояние входа;
- время входа.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/41.png "Действие входа")

Представление списка можно настроить, щелкнув **Столбцы** на панели инструментов.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/19.png "Действие входа")

Здесь вы можете добавить или удалить отображаемые поля.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/42.png "Действие входа")

Щелкнув элемент в представлении списка, вы получите дополнительные сведения о нем.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/43.png "Действие входа")


## <a name="filtering-sign-in-activities"></a>Фильтрация действий входа

Для сужения результатов до нужного уровня вы можете отфильтровать данные о входах, используя следующие поля:

- "Интервал времени	";
- Пользователь
- Приложение
- Клиент
- состояние входа.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/44.png "Действие входа")


Фильтр по **интервалу времени** позволяет определить интервал времени для возвращаемых данных.  
Возможные значения:

- 1 месяц
- 7 дней
- 24 часа
- Пользовательская

При выборе пользовательского интервала времени можно настроить время начала и окончания.

Фильтр по **пользователю** позволяет указать интересующее вас имя пользователя или имя участника-пользователя (UPN).

Фильтр по **приложению** позволяет указать имя интересующего вас приложения.

Фильтр по **клиенту** позволяет указать сведения об интересующем вас устройстве.

В фильтре по **состоянию входа** можно выбрать один из следующих фильтров:

- Все
- Успешно
- Сбой


## <a name="sign-in-activities-shortcuts"></a>Ярлыки для действий входа

Помимо Azure Active Directory, портал Azure предоставляет две дополнительные точки входа для данных о действиях входа:

- Пользователи и группы
- корпоративные приложения.


### <a name="users-and-groups-sign-ins-activities"></a>Действия входа для пользователей и групп

Информация, доступная в отчете о входе пользователя, поможет вам ответить на такие вопросы:

- Что такое шаблон входа пользователя?
- Сколько пользователей входили в течение недели?
- Каков статус их входа?



Знакомство с этими данными нужно начать с графика входов пользователей в разделе **Обзор**, который можно выбрать в колонке **Пользователи и группы**.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/45.png "Действие входа")

На графике входа еженедельно отображается количество входов всех пользователей за определенный промежуток времени, который по умолчанию составляет 30 дней.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/46.png "Действие входа")

Щелкнув день на графике входов, вы увидите подробный список действий входа за этот день.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/41.png "Действие входа")

Каждая строка в списке действий входа содержит подробную информацию о выбранном входе и отвечает на такие вопросы:

* Кто выполнил вход?
* Какое связанное имя участника-пользователя фигурировало?
* Какое приложение было целью входа?
* С какого IP-адреса выполнен вход?
* Каким был статус входа?

С помощью параметра **События входа** можно полностью отобразить все входы пользователей.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/51.png "Действие входа")



## <a name="usage-of-managed-applications"></a>Использование управляемых приложений

Представление данных входа, ориентированное на приложения, позволяет ответить на такие вопросы:

* Кто использует мои приложения?
* Какие три приложения являются самыми популярными в моей организации?
* Как обстоят дела с приложением, которое было недавно создано и развернуто?

Знакомство с этими данными нужно начать с трех приложений, которые в отчете за последние 30 дней являются самыми популярными (раздел **Обзор**, который можно выбрать в колонке **Корпоративные приложения**).

![Действие входа](./media/active-directory-reporting-activity-sign-ins/64.png "Действие входа")

В графике ниже (график использования приложений) отображены еженедельные входы в три самых популярных приложения за определенный промежуток времени, который по умолчанию составляет 30 дней.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/47.png "Действие входа")

Если нужно, вы можете переместить фокус на определенное приложение.


![Отчеты](./media/active-directory-reporting-activity-sign-ins/single_spp_usage_graph.png "Отчеты")

Щелкнув день на графике использования приложений, вы увидите подробный список действий входа.


![Действие входа](./media/active-directory-reporting-activity-sign-ins/48.png "Действие входа")


С помощью параметра **Входов** можно полностью отобразить все события входа в ваши приложения.

![Действие входа](./media/active-directory-reporting-activity-sign-ins/49.png "Действие входа")
