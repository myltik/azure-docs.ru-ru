# <a name="enable-subscription-management-in-your-tenant"></a>Включение управления подписками в клиенте

Глобальный администратор каталога может не иметь доступа по умолчанию ко всем ресурсам подписки в клиенте. В этой статье описана процедура для получения доступа ко всем подпискам в клиенте. В ней также предоставлен рекомендованный подход к обеспечению соответствия любым средствам безопасности, требуемых в организации после получения доступа.

## <a name="who-can-enable-management-of-subscriptions-in-my-directory"></a>Кто может включать управление подписками в каталоге?

Чтобы включить управление подпиской, каждый пользователь с ролью глобального администратора должен выполнить приведенные ниже шаги. Когда вы активируете управление подписками для себя, можно добавить других глобальных администраторов, которым также может требоваться доступ к ресурсам. Параметр каталога, который включает доступ для всех участников с ролью глобальный администратор, отсутствует.

## <a name="sign-in-to-the-azure-portal"></a>Выполните вход на портал Azure.

Войдите на портал Azure с учетной записью подходящего или активного участника роли глобального администратора. Если учетная запись принадлежит подходящему глобальному администратору, то роль надо активировать, перед тем как перейди к следующему этапу.

## <a name="access-the-azure-active-directory-admin-center"></a>Доступ к центру администрирования Azure Active Directory

После входа на портал Azure в качестве глобального администратора можно изменять параметры, чтобы получить доступ к подпискам Azure. Перейдите в центр администрирования Active Directory Azure (Azure AD) и выберите **Свойства**.

![Снимок экрана Azure AD центра администрирования с выделенным разделом "Свойства"](media/azure-pim-resource-rbac/aad_properties.png)

В списке свойств для параметра **Глобальный администратор может управлять подписками Azure** выберите **Да**.

![Снимок экрана страницы свойств со включенным параметром](media/azure-pim-resource-rbac/aad_properties_save.png)

Теперь ваша учетная запись будет автоматически добавлена к роли администратора доступа пользователей для каждого ресурса подписки в клиенте.

## <a name="browse-to-azure-ad-pim"></a>Переход к Azure AD PIM

 Далее перейдите к Azure AD Privileged Identity Management (PIM). В разделе **Управление** выберите **Ресурсы Azure**.

![Снимок экрана PIM с выделенными ресурсами Azure](media/azure-pim-resource-rbac/aadpim_manage_azure_resources.png)

## <a name="manage-and-discover-resources"></a>Управление и обнаружение ресурсов

Если организация уже использует Azure AD PIM для защиты администраторов в Azure AD, то при загрузке колонки будет отображен список подписок.

![Снимок экрана PIM со списком подписок в колонке](media/azure-pim-resource-rbac/aadpim_manage_azure_resource_some_there.png)

> [!NOTE]
> Если подписки не отображаются, убедитесь, что:
>- срок действия роли глобального администратора не истек; 
>- у организации есть подписка Azure.

![Снимок экрана PIM с пустым списком ресурсов](media/azure-pim-resource-rbac/aadpim_rbac_empty_resource_list.png)

## <a name="configure-assignments"></a>Настройка назначений

Выберите подписку из списка и назначьте себя (или группу, участником которой вы являетесь) в качестве соответствующего владельца ресурса. 
См. [дополнительные сведения о назначении ролей ](pim-resource-roles-assign-roles.md).

Перед тем как перейти к следующему этапу, повторите эту процедуру для каждого ресурса.

## <a name="clean-up-standing-access"></a>Удаление постоянного доступа

Настроив доступ к самым важным подпискам в организации, постоянный доступ можно удалить путем отключения соответствующего параметра в свойствах каталога.

![Снимок экрана страницы свойств со выключенным параметром](media/azure-pim-resource-rbac/aad_properties_no.png)
