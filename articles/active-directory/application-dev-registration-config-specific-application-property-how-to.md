# <a name="how-to-fill-out-specific-fields-for-a-custom-developed-application"></a>Заполнение определенных полей для специально разработанного приложения

В этой статье дается краткое описание доступных полей в форме регистрации приложения на [портале Azure](https://portal.azure.com).

## <a name="register-a-new-application"></a>Регистрация нового приложения

-   Чтобы зарегистрировать новое приложение, перейдите на [портал Azure](https://portal.azure.com).

-   В области навигации слева щелкните **Azure Active Directory**.

-   Выберите **Регистрация приложений** и щелкните **Добавить**.

-   Откроется форма регистрации приложения.

## <a name="fields-in-the-application-registration-form"></a>Поля в форме регистрации приложения


| Поле            | ОПИСАНИЕ                                                                              |
|------------------|------------------------------------------------------------------------------------------|
| ИМЯ             | Имя приложения. Оно должно содержать не менее четырех знаков.                |
| Тип приложения | **Веб-приложение или веб-API** — приложение, представляющее веб-приложение, веб-API или и то, и другое. 
| |**Собственное приложение** — это приложение, которое устанавливается на устройство или компьютер пользователя.           |
| URL-адрес входа      | URL-адрес, по которому пользователи могут войти для использования вашего приложения                                  |

Если вы заполнили указанные выше поля, приложение регистрируется на портале Azure и вы будете перенаправлены на страницу приложения. Если нажать кнопку **Параметры** в области приложения, откроется страница параметров с дополнительными полями для настройки приложения. В следующей таблице описаны все поля на странице параметров. Обратите внимание, что в зависимости от типа созданного приложения (веб-приложение или собственное приложение) будет отображаться всего несколько из этих полей.

| Поле           | ОПИСАНИЕ                                                                                                                                                                                                                                                                                                     |
|-----------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Идентификатор приложения  | При регистрации приложения Azure AD назначает ему идентификатор приложения. Его можно использовать для уникальной идентификации приложения в запросах проверки подлинности в Azure AD, а также для доступа к ресурсам, таким как API Graph.                                                          |
| URI идентификатора приложения      | Это должен быть уникальный URI, как правило, вида **https://&lt;имя\_клиента&gt;/&lt;имя\_приложения&gt;.** Он используется в потоке предоставления авторизации в качестве уникального идентификатора, указывающего ресурс, для которого должен быть выдан токен. Он также становится утверждением aud в выданном токене. |
| Отправка нового логотипа | С помощью этого элемента можно передать логотип для вашего приложения. Логотип должен иметь формат BMP, JPG или PNG. Размер файла должен быть менее 100 КБ. Изображение должно быть размером 215 x 215 пикселей с размером центрального изображения 94 x 94 пикселя.                                                       |
| URL-адрес домашней страницы   | Это URL-адрес входа, указанный при регистрации приложения.                                                                                                                                                                                                                                              |
| URL-адрес выхода      | Это URL-адрес единого выхода. Azure AD отправляет запрос на выход по этому URL-адресу, когда пользователь очищает сеанс с Azure AD с помощью любого другого зарегистрированного приложения.                                                                                                                                       |
| Мультитенантный  | Этот переключатель позволяет указать, может ли приложение использоваться несколькими клиентами. Обычно это означает, что внешние организации могут использовать ваше приложение, зарегистрировав его в своем клиенте и предоставив доступ к своим корпоративным данным.                                                                   |
| URL-адреса ответа      | URL-адреса ответа — это конечные точки, куда Azure AD возвращает все токены, запрашиваемые вашим приложением.                                                                                                                                                                                                          |
| URI перенаправления   | Для собственных приложений пользователь отправляется сюда после успешной авторизации. Azure AD проверяет, совпадает ли предоставленный приложением URI перенаправления в запросе OAuth 2.0 с одним из зарегистрированных значений на портале.                                                            |
| ключей            | Вы можете создать ключи для программного доступа к веб-API, защищенным Azure AD, без вмешательства пользователя. На странице \*\*Ключи\*\* введите описание ключа и дату истечения срока действия, а затем выполните сохранение для создания ключа. Сохраните ключ в надежном месте, так как вы не сможете получить к нему доступ позже.             |
