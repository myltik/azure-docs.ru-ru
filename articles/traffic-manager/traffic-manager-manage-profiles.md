# <a name="manage-an-azure-traffic-manager-profile"></a>Управление профилем диспетчера трафика Azure

Профили диспетчера трафика используют методы маршрутизации трафика, чтобы управлять распределением трафика для облачных служб и конечных точек веб-сайта. В статье объясняется, как создавать эти профили и управлять ими.

## <a name="create-a-traffic-manager-profile"></a>Создание профиля диспетчера трафика

Профиль диспетчера трафика можно создать с помощью портала Azure. После создания профиля вы сможете настроить на портале Azure конечные точки, мониторинг и другие параметры. Диспетчер трафика поддерживает до 200 конечных точек на профиль. Но для большинства сценариев использования требуется небольшое количество конечных точек.

### <a name="to-create-a-traffic-manager-profile"></a>Создание профиля диспетчера трафика

1. В браузере войдите на [портал Azure](http://portal.azure.com). Если у вас нет учетной записи, вы можете зарегистрироваться для получения [бесплатной пробной версии на один месяц](https://azure.microsoft.com/free/). 
2. Последовательно выберите **Создать ресурс** > **Сети** > **Профиль диспетчера трафика** > **Создать**.
4. В области **Создание профиля диспетчера трафика** сделайте следующее:
    1. В поле **Имя** укажите имя профиля. Оно должно быть уникальным в пределах зоны trafficmanager.net. В результате будет создано DNS-имя <name>, trafficmanager.net, которое будет использоваться для доступа к профилю диспетчера трафика.
    2. Из списка **Метод маршрутизации** выберите метод маршрутизации **Приоритет**.
    3. Из списка **Подписка** выберите подписку, в которой необходимо создать этот профиль.
    4. В разделе **Группа ресурсов** создайте группу ресурсов, в которую следует поместить этот профиль.
    5. Из списка **Расположение группы ресурсов** выберите расположение группы ресурсов. Этот параметр задает расположение группы ресурсов и не влияет на профиль диспетчера трафика, который будет развернут глобально.
    6. Нажмите кнопку **Создать**.
    7. После завершения глобального развертывания профиля диспетчера трафика он будет отображен в соответствующей группе ресурсов как один из ресурсов.

## <a name="disable-enable-or-delete-a-profile"></a>Включение, отключение или удаление профиля

Вы можете отключить существующий профиль диспетчера трафика, чтобы он не указывал запросам пользователей на настроенные конечные точки. При отключении профиля диспетчера трафика сам профиль и содержащиеся в нем сведения останутся нетронутыми и могут быть отредактированы в интерфейсе диспетчера трафика.  При повторном включении профиля перенаправление возобновляется. Создаваемый на портале Azure профиль диспетчера трафика включается автоматически. Если вы решили, что профиль больше не нужен, его можно удалить.

### <a name="to-disable-a-profile"></a>Отключение профиля

1. При использовании имени пользовательского домена измените запись CNAME на DNS-сервере в Интернете, чтобы она больше не указывала на профиль диспетчера трафика.
2. Трафик больше не будет направляться на конечные точки с помощью параметров профиля диспетчера трафика.
3. В браузере войдите на [портал Azure](http://portal.azure.com).
2. На панели поиска портала выполните поиск по имени **профиля диспетчера трафика**, которое необходимо изменить, а затем щелкните профиль диспетчера трафика в отображаемых результатах.
3. Последовательно выберите **Обзор** > **Отключить**.
4. Подтвердите, что собираетесь отключить профиль диспетчера трафика.

### <a name="to-enable-a-profile"></a>Включение профиля

1. В браузере войдите на [портал Azure](http://portal.azure.com).
2. На панели поиска портала выполните поиск по имени **профиля диспетчера трафика**, которое необходимо изменить, а затем щелкните профиль диспетчера трафика в отображаемых результатах.
3. Последовательно выберите **Обзор** > **Включить**.
1. При использовании имени пользовательского домена создайте запись ресурса CNAME на DNS-сервере в Интернете, указывающую на доменное имя вашего профиля диспетчера трафика.
2. Трафик снова будет направляться на конечные точки.

### <a name="to-delete-a-profile"></a>Порядок удаления профиля

1. Убедитесь, что запись ресурса DNS на DNS-сервере Интернета больше не использует запись ресурса CNAME, указывающую на доменное имя вашего профиля диспетчера трафика.
2. На панели поиска портала выполните поиск по имени **профиля диспетчера трафика**, которое необходимо изменить, а затем щелкните профиль диспетчера трафика в отображаемых результатах.
3. Последовательно выберите **Обзор** > **Удалить**.
4. Подтвердите, что собираетесь удалить профиль диспетчера трафика.
