# <a name="power-bi-embedded-faq"></a>Часто задаваемые вопросы о Power BI Embedded
## <a name="whats-your-most-recent-announcement-about-power-bi-embedded"></a>На какой стадии находится служба Power BI Embedded?
Уже выпущена общедоступная версия Microsoft Power BI Embedded со стандартным соглашением об уровне обслуживания. Дополнительные сведения об этом выпуске см. в статье [Новые возможности Power BI Embedded](power-bi-embedded-whats-new.md).

## <a name="what-is-microsoft-power-bi-embedded"></a>Что такое Microsoft Power BI Embedded?
Служба Microsoft Power BI Embedded теперь является общедоступной. Эта служба Azure позволяет разработчикам встраивать прекрасные, полностью интерактивные отчеты и визуализации в клиентские приложения, не тратя время и деньги на создание собственных элементов управления с нуля. Служба Power BI Embedded доступна с соглашением об уровне обслуживания в 9 центрах обработки данных по всему миру. Теперь она содержит усовершенствованные функциональные возможности, например поддержку защиты данных с применением безопасности на уровне строк для улучшенной фильтрации. Кроме того, мы упростили и обновили модель ценообразования Power BI Embedded. Дополнительные сведения см. на странице [Power BI Embedded](https://azure.microsoft.com/services/power-bi-embedded/).  

## <a name="who-would-want-to-use-microsoft-power-bi-embedded-and-why"></a>Кому следует использовать Microsoft Power BI Embedded и почему?
Служба Microsoft Power BI Embedded предназначена для разработчиков приложений, которые хотят предложить своим пользователям великолепные интерактивные возможности визуализации данных на любых устройствах без необходимости создавать визуализации самостоятельно. Со службой Power BI Embedded разработчики всегда могут предоставлять самые актуальные теории и мнения при помощи прямых запросов. Разработчики также могут программно разворачивать, автоматизировать Power BI и управлять ею с помощью API-интерфейсов Azure ARM и Power BI. Как и во всем, что касается Power BI, встроенная служба автоматически масштабируется в соответствии с использованием и потребностями вашего приложения. Служба Power BI Embedded использует модель ценообразования с оплатой по мере использования.

## <a name="how-does-power-bi-embedded-relate-to-the-power-bi-service"></a>Как служба Power BI Embedded связана со службой Power BI?
Служба Power BI Embedded и служба Power BI — это разные предложения. Служба Power BI Embedded использует модель ценообразования с оплатой по мере использования и развертывается на портале Azure. С ее помощью независимые поставщики программного обеспечения могут встраивать в свои приложения визуализации данных для использования клиентами. Служба Power BI тарифицируется и развертывается через портал O365. Служба Power BI — это автономное предложение BI общего назначения, предназначенное преимущественно для внутреннего корпоративного использования. Дополнительные сведения о службе Power BI см. на сайте [www.powerbi.com](https://powerbi.microsoft.com).

## <a name="how-does-power-bi-embedded-improve-my-app"></a>Как Power BI Embedded позволяет улучшить мое приложение?
Использование прекрасных интерактивных визуализаций данных прямо в приложении позволяет пользователям оперативно принимать решения и значительно улучшает приложение. Power BI Embedded позволяет улучшить приложение с помощью интерактивных и актуальных визуализаций данных с широкими возможностями. Это позволяет повысить удобство использования, удовлетворенность и верность пользователей и легко доставлять контекстную аналитику на любые устройства.

## <a name="are-there-any-rules-or-restrictions-about-how-i-can-use-power-bi-embedded-in-my-app"></a>Существуют ли все правила или ограничения, относящиеся к использованию Power BI Embedded в моем приложении?
Служба Power BI Embedded предназначена для приложений, предоставляемых внешним пользователям. Если вы хотите использовать службу Power BI Embedded для создания внутреннего бизнес-приложения, всем внутренним пользователям следует предоставить лицензию USL Power BI Pro. При этом помимо лицензионных платежей с вашей организации будет взиматься плата за использование службы Power BI Embedded. Чтобы помочь пользователям не платить ни за лицензию USL Power BI Pro, ни за использование службы Power BI Embedded для внутренних приложений, в службе Power BI предусмотрены собственные средства встраивания содержимого без Power BI Embedded, которые не требуют дополнительных затрат от владельцев подписки USL Power BI (dev.powerbi.com).

## <a name="can-power-bi-embedded-be-used-to-create-internal-applications"></a>Можно ли создавать внутренние приложения с помощью Power BI Embedded?
Нет. Решение Power BI Embedded предназначено для внешнего использования и не может применяться с внутренними бизнес-приложениями. Чтобы встроить содержимое Power BI для использования во внутренних бизнес-приложениях, вам нужно использовать службу Power BI. Все пользователи, которые используют это содержимое, должны иметь действительную лицензию на пользовательскую подписку Power BI (бесплатная версия) или Power BI Pro. Дополнительные сведения об интеграции внутренних приложений со службой Power BI см. на сайте [https://dev.powerbi.com](https://dev.powerbi.com).

## <a name="is-this-service-available-globally"></a>Эта служба доступна по всему миру?
Служба Power BI Embedded уже доступна в большинстве центров обработки данных. Актуальные сведения о доступности можно просмотреть [здесь](https://azure.microsoft.com/status/).

## <a name="what-is-the-available-sla-for-the-service"></a>Какое соглашение об уровне обслуживания доступно для службы?
Power BI Embedded со стандартным соглашением об уровне обслуживания Azure. Дополнительные сведения см. в на странице [соглашений об уровне обслуживания](https://azure.microsoft.com/support/legal/sla/).

Дополнительные сведения о ценах см. на странице с информацией о [ценах на Power BI Embedded](http://go.microsoft.com/fwlink/?LinkId=760527).

## <a name="what-is-a-report-session-and-how-is-it-billed"></a>Что такое сеанс отчетов и как за него выставляются счета?
Сеанс — это набор действий пользователя, совершаемых с отчетом Power BI Embedded. Сеанс инициируется при каждом отображении отчета Power BI Embedded пользователю. Счет за сеанс при этом выставляется владельцу подписки. Сеансы тарифицируются по фиксированной ставке независимо от количества визуальных элементов в отчете или частоты обновления его содержимого. Сеанс завершается, когда пользователь закрывает отчет, или по истечении времени ожидания сеанса — через час.

## <a name="do-you-offer-any-tools-or-guidance-to-help-me-estimate-how-many-renderssession-i-should-expect-how-will-i-know-how-many-renders-have-been-completed"></a>Существуют ли какие-то инструменты или рекомендации, которые помогут оценить предполагаемое количество отрисовок или сеансов? Как узнать о том, сколько отрисовок выполнено?
Сведения о выставлении счетов в соответствии с количеством отрисовок или сеансов отчетов, выполненных для вашей подписки, отображаются на портале Azure.

## <a name="do-i-need-a-power-bi-subscription-in-order-to-develop-applications-with-power-bi-embedded-how-do-i-get-started"></a>Необходима ли подписка Power BI для разработки приложений с Power BI Embedded? Как начать работу?
Для создания отчетов и визуализаций, которые будут использоваться в приложении, вам как разработчику приложения подписка Power BI не требуется. Вам потребуются подписка Microsoft Azure и бесплатное приложение Power BI Desktop.

Сведения о том, как использовать службу Power BI Embedded, см. в документации по нашей службе.

## <a name="i-have-an-azure-subscription-can-i-use-power-bi-embedded-using-my-existing-subscription"></a>У меня есть подписка Azure. Можно ли использовать Power BI Embedded с имеющейся подпиской?
Да. Для подготовки и использования службы Microsoft Power BI Embedded можно использовать существующую подписку Azure.

## <a name="does-my-application-end-user-need-a-power-bi-license"></a>Нужна ли лицензия Power BI конечным пользователям моего приложения?
Нет. Конечным пользователям вашего приложения не требуется приобретать отдельную подписку Power BI для доступа к визуализациям данных в приложении. В модели Power BI Embedded счет за использование службы выставляется вам как поставщику приложения с помощью индикатора потребления Azure. Дополнительные сведения см. на [странице сведений о ценах и лицензировании](http://go.microsoft.com/fwlink/?LinkId=760527).

## <a name="how-does-user-authentication-work-with-power-bi-embedded"></a>Как выполняется проверка подлинности пользователя в Power BI Embedded?
Служба Power BI Embedded вместо явной проверки подлинности пользователей использует маркеры приложений для проверки подлинности и авторизации. В модели с маркерами приложений проверкой подлинности и авторизацией конечных пользователей управляет ваше приложение. При необходимости ваше приложение создает

и отправляет маркеры приложений, которые сообщают службе о том, что необходимо подготовить запрошенный отчет. Этот подход не требует использования Azure AD для проверки подлинности и авторизации пользователей, хотя вы можете это сделать. Дополнительные сведения о маркерах приложений см. [здесь](power-bi-embedded-app-token-flow.md). Для расширенных сценариев фильтрации безопасности в Power BI Embedded представлена функция безопасности на уровне строк.

## <a name="what-data-sources-are-currently-supported-with-power-bi-embedded"></a>Какие источники данных в настоящее время поддерживаются Power BI Embedded?
Будет поддерживаться доступ к облачным источникам данных, которые используют базовые учетные данные через прямые запросы. Это означает, что прямо сейчас поддерживаются такие источники данных, как база данных SQL Azure и хранилище данных SQL Azure. Мы добавим поддержку других источников данных и типов доступа в ближайшие месяцы. Дополнительные сведения см. в разделе [Подключение к источнику данных](power-bi-embedded-connect-datasource.md).

## <a name="how-does-the-tenancy-model-work-for-power-bi-embedded"></a>Как работает клиентская модель для Power BI Embedded?
В модели Power BI Embedded нет явной необходимости в том, чтобы ваши клиенты были клиентами Azure AD. Вы можете потребовать от своих клиентов использовать Azure AD, а можете и не делать этого. В результате клиентская модель, необходимая для Power BI Embedded, будет определяться архитектурой и инфраструктурой вашего приложения.

Разработчикам и сотрудникам, работающим над созданием вашего приложения, потребуется учетная запись AAD для управления подпиской Azure и коллекциями рабочей области на портале Azure. Программные интерфейсы API для импорта отчетов, изменения строк подключения и получения URL-адресов встраивания для проверки подлинности используют маркеры приложения и поэтому не требуют AAD. Сведения о том, как использовать интерфейсы API и портал Azure, можно найти [на странице документации по Power BI Embedded ](https://azure.microsoft.com/documentation/services/power-bi-embedded/) на сайте Azure.com.

## <a name="how-do-i-get-started"></a>Как начать работу?
Сейчас вы можете начать работу бесплатно! Если у вас есть подписка Azure, то вы сможете подготовить Power BI Embedded прямо на портале Azure.  Вы также можете создать [бесплатную учетную запись Azure](https://azure.microsoft.com/free/). После подготовки службы Power BI Embedded вы сможете легко применить интерфейсы REST API Power BI напрямую или использовать пакет SDK для разработчиков, который доступен на сайте [GitHub](http://go.microsoft.com/fwlink/?LinkID=746472). В нем также представлены примеры по использованию пакета SDK для разработчиков.
