# <a name="monitor-identity-and-access-in-azure-security-center"></a>Мониторинг идентификации и доступа в центре безопасности Azure
Из этой статьи вы узнаете, как с помощью центра безопасности Azure отслеживать действия, связанные с идентификацией и доступом пользователей.

## <a name="why-monitor-identity-and-access"></a>Зачем нужен мониторинг идентификации и доступа?
Для предприятия идентификация должна быть плоскостью управления, а защита удостоверений — основным приоритетом. Раньше вокруг организаций существовали периметры, которые были одной из основных защитных границ. В настоящее время, когда большинство данных и приложений перемещаются в облако, идентификация становится новым периметром.

Отслеживая действия по идентификации, можно принимать упреждающие меры до возникновения инцидента или ответные меры для предотвращения попыток атак. Панель мониторинга идентификации и доступа предоставляет обзор состояния идентификации, включая:

* число неудачных попыток входа в систему; 
* учетные записи пользователей, которые использовались во время этих попыток;
* учетные записи, которые были заблокированы;
* учетные записи, для которых был изменен или сброшен пароль; 
* текущее число учетных записей, которые выполнили вход.

## <a name="monitor-identity-and-access-activities"></a>Мониторинг действий, связанных с доступом и идентификацией
Чтобы получить доступ к панели мониторинга **Identity & Access** (Идентификация и доступ) для просмотра текущих действий, связанных с идентификацией и доступом, сделайте следующее.

1. Откройте панель мониторинга **центра безопасности**.

2. В области слева в разделе **Предотвращение** выберите **Identity & Access** (Идентификация и доступ). При наличии нескольких рабочих областей появится селектор рабочих областей.

    ![Выбор рабочей области](./media/security-center-identity-access\security-center-identity-access-fig1.png)

    > [!NOTE]
    > Если в столбце справа отображается **UPGRADE PLAN** (План обновления), рабочая область использует бесплатную подписку. Повысьте уровень до стандартной подписки для использования этой функции. Если в столбце справа отображается **REQUIRES UPDATE** (Требуется обновить), обновите [Azure Log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-overview) для использования этой функции. Дополнительные сведения о тарифном плане см. на странице с ценами на центр безопасности. 
    > 
3. Если вам нужно изучить более одной рабочей области, вы можете расставить приоритеты в соответствии со столбцом **Ошибки входа**. В нем отображается текущее количество неудачных попыток входа в этой рабочей области. Щелкните рабочую область, которую необходимо использовать. Появится панель мониторинга **Identity & Access** (Идентификация и доступ).

    ![Идентификация и доступ](./media/security-center-identity-access\security-center-identity-access-fig2.png)

4. Сведения, доступные на этой панели мониторинга, могут помочь вам мгновенно определить потенциально подозрительную активность. Эта панель мониторинга состоит из трех основных областей:

    a. **Состояние идентификатора**. Предоставляет сводные данные о действиях идентификации, выполненных в этой рабочей области.

    Б. **Ошибки входа**. Помогает быстро определить основную причину ошибок входа. Здесь отображается список первых 10 учетных записей, которым не удалось выполнить большинство попыток входа в систему.

    c. **Входы в систему в динамике по времени**. Помогает быстро определить число входов в систему в динамике по времени. Здесь отображается список основных попыток входа в систему с помощью компьютера.
    
Независимо от того, какой элемент вы выберете, панель инструментов, которая появится, зависит от запроса поиска по журналам. Единственным различием является тип запроса и результат. Вы по-прежнему можете выбрать элемент, например компьютер, и увидеть соответствующие данные.
