# <a name="azure-security-center-and-azure-sql-database-service"></a>Центр безопасности Azure и служба базы данных SQL Azure
[Центр безопасности Azure](https://azure.microsoft.com/documentation/services/security-center/) позволяет предотвращать, обнаруживать угрозы и реагировать на них. Она включает встроенные функции мониторинга безопасности и управления политиками для подписок Azure, помогает выявлять угрозы, которые в противном случае могли бы оказаться незамеченными, и взаимодействует с широким комплексом решений по обеспечению безопасности.

В этой статье показано, как центр безопасности помогает защитить базы данных в Базе данных SQL Azure.

## <a name="why-use-security-center"></a>Для чего используется центр безопасности?
Центр безопасности помогает защитить данные в базе данных SQL, обеспечивая представление безопасности всех серверов и баз данных. Ниже перечислены возможности центра безопасности:

* определение политик для шифрования и аудита базы данных SQL;
* мониторинг безопасности ресурсов базы данных SQL во всех ваших подписках;
* быстрое выявление и устранение проблем безопасности;
* интеграция оповещений об [обнаружении угроз для базы данных SQL Azure](../sql-database/sql-database-threat-detection.md).

Помимо защиты ресурсов базы данных SQL, центр безопасности предоставляет средства мониторинга и управления безопасностью виртуальных машин Azure, облачных служб, служб приложений, виртуальных сетей, а также многие другие возможности. Дополнительные сведения о центре безопасности можно получить [здесь](security-center-intro.md).

## <a name="prerequisites"></a>предварительным требованиям
Для начала работы с Центром безопасности необходима подписка Microsoft Azure. Уровень "Бесплатный" центра безопасности включен в вашей подписке. Дополнительные сведения об уровнях "Бесплатный" и "Стандартный" центра безопасности см. в разделе [Центр безопасности — цены](https://azure.microsoft.com/pricing/details/security-center/).

Центр безопасности поддерживает доступ на основе ролей. Дополнительные сведения об управлении доступом на основе ролей (RBAC) в Azure см. в статье [Контроль доступа на основе ролей в Azure Active Directory](../role-based-access-control/role-assignments-portal.md). Вопросы и ответы по центру безопасности позволят узнать об [управлении разрешениями в центре безопасности](security-center-faq.md#permissions).

## <a name="access-security-center"></a>Доступ к Центру безопасности
Доступ к Центру безопасности осуществляется через [портал Azure](https://azure.microsoft.com/features/azure-portal/). [Войдите портал](https://portal.azure.com/) и выберите **Центр безопасности**.

![Пункт "Центр безопасности"][1]

Откроется колонка **Центр безопасности**.
![Колонка "Центр безопасности"][2]

## <a name="set-security-policy"></a>Установка политики безопасности
Политика безопасности определяет набор элементов управления, которые рекомендуются для ресурсов в указанной подписке или группе ресурсов. В центре безопасности можно настраивать политики для подписок или групп ресурсов в соответствии с потребностями безопасности вашей компании, типом приложений или конфиденциальностью данных в каждой подписке.

Можно установить политику, чтобы показывать рекомендации по аудиту SQL и прозрачному шифрованию данных (TDE) SQL.

* При включении **аудита и обнаружения угроз SQL** центр безопасности рекомендует включить аудит доступа к базе данных Azure в целях обеспечения соответствия требованиям, расширенного обнаружения и исследования.
* При включении **прозрачного шифрования данных SQL** центр безопасности рекомендует включить шифрование неактивных данных для Базы данных SQL Azure, связанных резервных копий и файлов журналов транзакций.

Чтобы установить политику безопасности, щелкните элемент **Политика** в колонке "Центр безопасности". В колонке **Политика безопасности** выберите подписку, для которой необходимо включить политику безопасности. В колонке **Политика предотвращения** **включите** необходимые рекомендации по безопасности для этой подписки.
![Политика безопасности][3]

Дополнительные сведения см. в разделе [Установка политики безопасности](security-center-policies.md).

## <a name="manage-security-recommendation"></a>Управление рекомендациями по безопасности
Центр безопасности периодически анализирует состояние безопасности ресурсов Azure. Когда Центр безопасности выявляет потенциальные уязвимости системы безопасности, он создает рекомендации. Рекомендации помогают настраивать необходимые элементы управления.

После настройки политики безопасности центр безопасности анализирует состояние безопасности ресурсов, чтобы выявить потенциальные уязвимости. Рекомендации отображаются в виде таблицы, где каждая строка представляет одну конкретную рекомендацию. Ниже приведен пример такой таблицы, в которой показано, какие полезные рекомендации для базы данных SQL Azure доступны и каково их назначение. Если выбрать рекомендацию, то отобразится статья о том, как реализовать эту рекомендацию в центре безопасности.

| Рекомендации | ОПИСАНИЕ |
| --- | --- |
| [Включение аудита и обнаружения угроз на серверах SQL Server](security-center-enable-auditing-on-sql-servers.md) |Рекомендуется включить аудит и обнаружение угроз для серверов базы данных SQL. (Только для службы Базы данных SQL. Не затрагивает Microsoft SQL Server на ваших виртуальных машинах.) |
| [Включение аудита и обнаружения угроз для баз данных SQL](security-center-enable-auditing-on-sql-databases.md) |Рекомендуется включить аудит и обнаружение угроз для баз данных Базы данных SQL. (Только для службы Базы данных SQL. Не затрагивает Microsoft SQL Server на ваших виртуальных машинах.) |
| [Включение прозрачного шифрования данных](security-center-enable-transparent-data-encryption.md) |Рекомендует включить шифрование для баз данных SQL. (Только для службы Базы данных SQL.) |

Чтобы просмотреть рекомендации для ресурсов Azure, выберите элемент **Рекомендации** в колонке "Центр безопасности". В колонке **Рекомендации** щелкните рекомендацию, чтобы отобразить подробные сведения. В этом примере выберем рекомендацию **Включение аудита и обнаружения угроз на серверах SQL Server**.

![Рекомендации][4]

Как показано ниже, центр безопасности показывает серверы SQL Server, на которых не включены аудит и обнаружение угроз. После включения аудита можно настроить параметры обнаружения угроз и параметры электронных сообщений для получения оповещений системы безопасности. Система обнаружения угроз оповещает об аномальных действиях в базе данных, указывающих на наличие потенциальных угроз безопасности. Оповещения отображаются на панели мониторинга центра безопасности.
![Аудит и обнаружение угроз][5]

Следуйте указаниям в разделе [Обнаружение угроз для базы данных SQL](../sql-database/sql-database-threat-detection-portal.md), чтобы включить и настроить обнаружение угроз, а также настроить список электронных адресов, на которые будут отправляться оповещения системы безопасности при обнаружении аномальных действий.

Дополнительные сведения о рекомендациях см. в разделе [Управление рекомендациями по безопасности в Центре безопасности Azure](security-center-recommendations.md).

## <a name="monitor-security-health"></a>Отслеживание состояния системы безопасности
Когда вы включаете [политики безопасности](security-center-policies.md) для ресурсов в рамках подписки, центр безопасности проверяет состояние защиты ресурсов, чтобы определить потенциальные уязвимости.  Сведения о состоянии безопасности ваших ресурсов можно просмотреть в элементе **Работоспособность безопасности ресурсов**. Если в элементе **Работоспособность безопасности ресурсов** щелкнуть **Данные**, откроется колонка **Data Resources** (Ресурсы данных). Она содержит рекомендации для SQL по устранению таких проблем, как отключенный аудит и отключенное прозрачное шифрование данных. Она также содержит рекомендации, касающиеся общего состояния работоспособности базы данных.
![Работоспособность безопасности ресурсов][6]

Дополнительные сведения см. в разделе [Наблюдение за работоспособностью системы безопасности в Центре безопасности Azure](security-center-monitoring.md).

## <a name="manage-and-respond-to-security-alerts"></a>Управление оповещениями системы безопасности и реагирование на них
Центр безопасности автоматически собирает, анализирует и объединяет данные журналов из [системы обнаружения угроз SQL Azure](../sql-database/sql-database-threat-detection.md) и других ресурсов для выявления реальных угроз и сокращения количества ложных срабатываний. Список приоритетных оповещений системы безопасности отображается в центре безопасности вместе с информацией, необходимой для быстрого анализа проблемы, и рекомендациями по устранению атаки.

Чтобы просмотреть оповещения, вернитесь к элементу **Оповещения системы безопасности** в колонке "Центр безопасности". В колонке **Оповещения системы безопасности** выберите оповещение, чтобы узнать больше о событиях, активировавших оповещение, и (при необходимости) действиях, которые следует предпринять для устранения атаки. В этом примере выберем **потенциальное внедрение кода SQL**.
![Оповещения системы безопасности][7]

Как показано ниже, центр безопасности предоставляет дополнительные сведения, которые содержат причину активации оповещения системы безопасности, целевой ресурс, IP-адрес источника (если применимо) и рекомендации по исправлению.
![Потенциальная атака путем внедрения кода SQL][8]

Дополнительные сведения см. в разделе [Управление оповещениями безопасности в Центре безопасности Azure и реагирование на них](security-center-managing-and-responding-alerts.md).
