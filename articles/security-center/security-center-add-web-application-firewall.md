# <a name="add-a-web-application-firewall-in-azure-security-center"></a>Добавление брандмауэра веб-приложения в Центре безопасности Azure
Центр безопасности Azure может рекомендовать добавить брандмауэр веб-приложения (WAF) от партнера корпорации Майкрософт в целях обеспечения безопасности веб-приложений. В этом документе вы найдете пример применения данной рекомендации.

Рекомендация развернуть WAF отображается для любого общедоступного IP-адреса (IP-адреса уровня экземпляра или IP-адреса с балансировкой нагрузки), имеющего связанную группу безопасности сети с открытыми входящими веб-портами (80, 443).

Центр безопасности рекомендует подготовить WAF для защиты от атак, направленных на веб-приложения на виртуальных машинах и в среде службы приложений. Среда службы приложений (ASE) — это параметр плана обслуживания [Премиум](https://azure.microsoft.com/pricing/details/app-service/) в службе приложений Azure. Она обеспечивает полностью изолированную и выделенную среду для безопасного выполнения приложений службы приложений Azure. Дополнительные сведения об ASE см. в [документации по среде службы приложений](../app-service/environment/intro.md).

> [!NOTE]
> В документе приводится обзор службы с помощью примера развертывания.  Этот документ не является пошаговым руководством.
>
>

## <a name="implement-the-recommendation"></a>Выполнение рекомендаций
1. В колонке **Рекомендации** выберите рекомендацию **Secure web application using web application firewall** (Защита веб-приложения с помощью брандмауэра веб-приложения).
   ![Защищенное веб-приложение][1]
2. В колонке **Защита веб-приложений с помощью брандмауэра веб-приложения** выберите веб-приложение. Откроется колонка **Добавление брандмауэра веб-приложения** .
   ![Add a web application firewall][2]
3. Можно использовать существующий брандмауэр веб-приложения (если он доступен) или создать новый. В этом примере нет доступных WAF, поэтому мы его создадим.
4. Чтобы создать WAF, выберите решение из списка решений партнеров по интеграции. В этом примере выберите **Barracuda Web Application Firewall**.
5. Откроется колонка **Barracuda Web Application Firewall** , в которой содержатся сведения о партнерском решении. В колонке сведений щелкните **Создать** .

   ![Колонка со сведениями о брандмауэре][3]

6. Откроется колонка **Новый брандмауэр веб-приложения**, где вы можете выполнить действия по **настройке виртуальной машины** и указать **сведения о WAF**. Выберите **Конфигурация виртуальной машины**.
7. В колонке **Конфигурация виртуальной машины** введите сведения, необходимые для настройки виртуальной машины, на которой выполняется WAF.
   ![VM configuration][4]
8. Вернитесь в колонку **Новый брандмауэр веб-приложения** и выберите **Сведения о WAF**. В колонке **Сведения о WAF** настраивается сам брандмауэр веб-приложения. На шаге 7 выполняется настройка виртуальной машины, на которой выполняется WAF, а на шаге 8 осуществляется подготовка самого брандмауэра.

## <a name="finalize-application-protection"></a>Завершение подготовки защиты приложений
1. Вернитесь в колонку **Рекомендации** . После создания WAF появляется новая запись с именем **Завершение подготовки защиты приложений**. Эта запись указывает, что необходимо завершить процедуру, фактически подключив брандмауэр WAF к виртуальной сети Azure, чтобы он мог защищать приложение.

   ![Завершение подготовки защиты приложений][5]

2. Выберите **Завершение подготовки защиты приложений**. Откроется новая панель. Вы увидите, что появилось веб-приложение, для которого нужно перенаправлять трафик.
3. Выберите веб-приложение. Откроется колонка, в которой приведены пошаговые инструкции по завершению настройки брандмауэра веб-приложения. Выполните эти действия и щелкните **Ограничить трафик**. Затем центр безопасности выполняет необходимые подключения.

   ![Ограничение трафика][6]

> [!NOTE]
> Вы можете защитить несколько веб-приложений в центре обеспечения безопасности, добавив эти приложения в существующие развернутые службы WAF.
>
>

Журналы WAF теперь полностью интегрированы. Центр безопасности может автоматически начать сбор и анализ журналов, чтобы создавать важные оповещения системы безопасности.
