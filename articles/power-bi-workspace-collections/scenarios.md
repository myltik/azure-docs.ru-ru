# <a name="common-power-bi-workspace-collection-scenarios"></a>Распространенные сценарии работы с коллекциями рабочих областей Power BI

Коллекции рабочих областей Microsoft Power BI предоставляют большинство существующих функций API Power BI как часть службы Azure. С помощью этих функций вы можете разрабатывать свои приложения.  Вы также можете с помощью программных средств подготавливать, разрабатывать и развертывать необходимые ресурсы и содержимое Power BI. См. дополнительные сведения о [начале работы с коллекциями рабочих областей Microsoft Power BI](get-started.md).

> [!IMPORTANT]
> Использовать службу "Коллекции рабочих областей Power BI" не рекомендуется. Она будет доступна до июня 2018 года или до даты, указанной в вашем договоре. Мы советуем организовать перенос приложения в Power BI Embedded, чтобы избежать прерываний в его работе. Сведения о том, как перенести данные из коллекций рабочих областей Power BI в Power BI Embedded, см. в [этой статье](https://powerbi.microsoft.com/documentation/powerbi-developer-migrate-from-powerbi-embedded/).

Ниже приведено несколько сценариев использования **коллекций рабочих областей Power BI** в приложениях.

* Полностью интерактивные отчеты, которые были созданы в Power BI Desktop, можно внедрить в приложение с помощью новой модели аутентификации приложений посредством маркера.
* Независимые поставщики программного обеспечения и его пользователи создают приложения для клиентов, которые могут использовать **коллекции рабочих областей Power BI** и пакет SDK для Power BI, чтобы внедрять интерактивные отчеты. См. дополнительные сведения о [начале работы с коллекциями рабочих областей Microsoft Power BI](get-started.md).
* Разработчик может использовать платформу визуализации Power BI для создания пользовательских визуализаций, которые можно применять в собственном приложении. См. страницу [Расширьте Power BI с помощью настраиваемых визуальных элементов](https://powerbi.microsoft.com/custom-visuals/).
