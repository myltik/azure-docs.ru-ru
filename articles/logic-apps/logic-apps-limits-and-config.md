# <a name="logic-app-limits-and-configuration"></a>Ограничения и настройка приложений логики

Ниже приведены сведения о текущих ограничениях и сведения о конфигурации Azure Logic Apps.

## <a name="limits"></a>Ограничения

### <a name="http-request-limits"></a>Ограничения HTTP-запросов

Ниже представлены ограничения для одного вызова HTTP-запроса и/или вызова соединителя.

#### <a name="timeout"></a>Время ожидания

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Истекло время ожидания запроса|120 секунд|При необходимости возможна компенсация с помощью [асинхронной схемы](../logic-apps/logic-apps-create-api-app.md) или [цикла Until](logic-apps-loops-and-scopes.md).|

#### <a name="message-size"></a>Размер сообщения

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Размер сообщения|100 МБ|Некоторые соединители и API могут не поддерживать размер 100 МБ |
|Оценочное ограничение выражения|131 072 символов|`@concat()`, `@base64()`, `string` не могут быть длиннее этого ограничения|

#### <a name="retry-policy"></a>Политика повтора

|Name (Имя)|Ограничение|Примечания|
|----|----|----|
|Количество повторных попыток.|10| Значение по умолчанию — 4. Можно настроить с помощью [параметра политики повтора](https://msdn.microsoft.com/en-us/library/azure/mt643939.aspx)|
|Максимальный интервал повторения|1 час|Можно настроить с помощью [параметра политики повтора](https://msdn.microsoft.com/en-us/library/azure/mt643939.aspx)|
|Минимальный интервал повторения|5 секунд|Можно настроить с помощью [параметра политики повтора](https://msdn.microsoft.com/en-us/library/azure/mt643939.aspx)|

### <a name="run-duration-and-retention"></a>Длительность выполнения и удержания

Ниже представлены ограничения для единого выполнения приложения логики.

|Имя|Ограничение|Примечания|
|----|----|----|
|Длительность выполнения|90 дней||
|Хранение периода удержания|90 дней|Считается со времени начала выполнения|
|Минимальный интервал повторения|1 с|| 15 секунд для приложений логики с планом службы приложений
|Максимальный интервал повторения|500 дней||

Если вы предполагаете, что превысите ограничения длительности выполнения и хранения при обычной последовательности обработки, [свяжитесь с нами](mailto://logicappsemail@microsoft.com). Мы поможем удовлетворить ваши требования.


### <a name="looping-and-debatching-limits"></a>Ограничения зацикливания и индивидуальной обработки

Ниже представлены ограничения для единого выполнения приложения логики.

|Имя|Ограничение|Примечания|
|----|----|----|
|Элементы ForEach|100 000|При необходимости можно использовать [действие запроса](../connectors/connectors-native-query.md) для фильтрации больших массивов|
|Максимальное количество итераций|5 000||
|Элементы SplitOn|100 000||
|Параллелизм ForEach|50| Значение по умолчанию — 20. Можно задать последовательность элементов ForEach, добавив `"operationOptions": "Sequential"` к действию `foreach` или определенный уровень параллелизма, используя `runtimeConfiguration`|


### <a name="throughput-limits"></a>Ограничения пропускной способности

Ниже представлены ограничения для одного экземпляра приложения логики. 

|Имя|Ограничение|Примечания|
|----|----|----|
|Выполняемые действия за 5 минут |100 000|При необходимости можно распределить рабочую нагрузку между несколькими приложениями|
|Исходящие параллельные вызовы действий |около 2500|При необходимости можно уменьшить количество параллельных запросов или длительность|
|Входящие параллельные вызовы конечной точки среды выполнения |~1000|При необходимости можно уменьшить количество параллельных запросов или длительность|
|Вызовы read конечной точки среды выполнения за 5 минут |60 000|При необходимости можно распределить рабочую нагрузку между несколькими приложениями|
|Вызовы invoke конечной точки среды выполнения за 5 минут |45 000|При необходимости можно распределить рабочую нагрузку между несколькими приложениями|

Если вы ожидаете, что это ограничение будет превышено при обычной обработке, или хотите запустить нагрузочное тестирование, при котором возможно превышение этого ограничения в течение заданного периода времени, [свяжитесь с нами](mailto://logicappsemail@microsoft.com), чтобы мы могли помочь выполнить ваши требования.
