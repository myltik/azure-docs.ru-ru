# <a name="encryption-requirements-for-pci-dss-compliant-environments"></a>Требования к шифрованию для сред, соответствующих стандарту PCI DSS 
## <a name="pci-dss-requirement-4"></a>Стандарт PCI DSS: требование 4

**Шифрование данных о владельцах карт при передаче по открытым общедоступным сетям**

> [!NOTE]
> Эти требования определены [Советом по стандартам безопасности индустрии платежных карт (Payment Card Industry Security Standards Council, PCI SSC)](https://www.pcisecuritystandards.org/pci_security/) в [стандарте PCI DSS версии 3.2](https://www.pcisecuritystandards.org/document_library?category=pcidss&document=pci_dss). Сведения о процедурах тестирования и рекомендации для каждого требования см. в документации стандарта PCI DSS.

При передаче по сетям, легкодоступным для злоумышленников, конфиденциальные сведения должны быть зашифрованы. Неправильно настроенные беспроводные сети и уязвимости в устаревших протоколах шифрования и проверки подлинности по-прежнему остаются мишенями для злоумышленников, которые используют эти уязвимости, чтобы получить привилегированный доступ к средам, где хранятся данные о владельцах карт.

## <a name="pci-dss-requirement-41"></a>Стандарт PCI DSS: требование 4.1

**4.1.** Использовать стойкое шифрование и безопасные протоколы (например, TLS, IPSEC, SSH и т. д.), чтобы защитить конфиденциальные данные владельцев карт при их передаче через открытые общедоступные сети с учетом следующего:
- принимаются только доверенные ключи и сертификаты;
- используемый протокол поддерживает только безопасные версии и конфигурации;
- стойкость шифрования соответствует используемой методологии шифрования. 

> [!NOTE]
> Там, где используется SSL и (или) ранние версии TLS, должны соблюдаться требования, описанные в Приложении А2.
>
> К примерам открытых общедоступных сетей в частности относятся:
> - сеть Интернет;
> - беспроводные технологии, включая протоколы 802.11 и Bluetooth;
> - технологии сотовой связи, например GSM и CDMA;
> - GPRS;
> - спутниковые средства связи.

### <a name="pci-dss-requirement-411"></a>Требование PCI DSS 4.1.1

**4.1.1.** Гарантировать, что при использовании беспроводных сетей, передающих данные владельцев карт либо подключенных к среде, в которой хранятся данные владельцев карт, применяются лучшие отраслевые методики (например, стандарт IEEE 802.11i), чтобы реализовать стойкое шифрование при аутентификации и передаче данных.

**Обязанности:&nbsp;&nbsp;`Customer Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Не применяется |
| **Клиент<br />(план PCI-DSS&nbsp;)** | Беспроводная связь и SNMP в этом решении не поддерживаются.|



## <a name="pci-dss-requirement-42"></a>Стандарт PCI DSS: требование 4.2

**4.2.** Запрещено пересылать незащищенные номера основных счетов (PAN) с использованием технологий обмена сообщениями для конечных пользователей (электронной почты, систем мгновенного обмена сообщениями, SMS, чатов и т. д.).

**Обязанности:&nbsp;&nbsp;`Customer Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Не применяется |
| **Клиент<br />(план PCI-DSS&nbsp;)** | В интернет-магазине Contoso не реализованы решения для обмена сообщениями, с помощью которых можно отправлять незащищенные данные PAN.|



## <a name="pci-dss-requirement-43"></a>Стандарт PCI DSS: требование 4.3

**4.3.** Гарантировать, что политики безопасности и операционные процедуры шифрования передаваемых данных владельцев карт задокументированы, используются и известны всем заинтересованным лицам.

**Обязанности:&nbsp;&nbsp;`Customer Only`**

|||
|---|---|
| **Поставщик<br />(Microsoft&nbsp;Azure)** | Не применяется |
| **Клиент<br />(план PCI-DSS&nbsp;)** | Клиенты несут ответственность за документирование и шифрование передаваемых данных владельцев карт.|
