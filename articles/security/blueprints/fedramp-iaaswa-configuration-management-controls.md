# <a name="configuration-management-cm"></a>Управление конфигурацией (CM)

> [!NOTE]
> Эти директивы определены NIST и Министерством торговли США в составе специальной публикации NIST 800-53 редакции 4. Сведения о процедурах тестирования и рекомендации по каждой директиве см. в документе NIST 800-53 редакции 4.

## <a name="nist-800-53-control-cm-1"></a>Директива CM-1 (NIST 800-53)

#### <a name="configuration-management-policy-and-procedures"></a>Политика и процедуры управления конфигурацией

**CM-1**. Организация разрабатывает, документирует и распространяет [определяемым организацией сотрудникам и ролям] политику управления конфигурацией, которая охватывает назначение, сферу действия, роли, обязанности, обязательства по управлению, координацию между организационными подразделениями и соответствие; процедуры для содействия осуществлению политики управления конфигурацией и соответствующих средств управления конфигурацией; рассматривает и обновляет текущую политику управления конфигурацией [с определенной организацией частотой]; процедуры обеспечения управления конфигурацией [с определенной организацией частотой].

**Обязанности:** только клиент (`Customer Only`)

|||
|---|---|
| **Клиент** | Политики и процедур управления конфигурацией на уровне предприятия клиента может быть достаточно для реализации этой директивы. |
| **Поставщик (Microsoft Azure)** | Не применяется |


 ## <a name="nist-800-53-control-cm-2"></a>Директива CM-2 (NIST 800-53)

#### <a name="baseline-configuration"></a>Базовая конфигурация

**CM-2**. Организация разрабатывает, документирует и сохраняет в рамках управления конфигурацией текущую базовую конфигурацию информационной системы.

**Обязанности:** только клиент (`Customer Only`)

|||
|---|---|
| **Клиент** | Шаблоны Azure Resource Manager и соответствующие ресурсы, которые входят в эту схему, представляют собой базовый план "конфигурация как код" для развернутой архитектуры. Решение предоставляется через GitHub, который можно использовать для управления конфигурацией. Решение содержит базовый план Desired State Configuration для каждой развернутой виртуальной машины. |
| **Поставщик (Microsoft Azure)** | Не применяется |
